## the build

`docs/realm/js/ui.js` — single block in
`updateUI()` near the existing raid-warning,
+ one HTML interpolation update.

```js
let streakHTML = '';
if (!raidDays && G.stats?.raidsSurvived >= 1 && G.lastRaidDay !== undefined) {
  const peaceD = G.day - G.lastRaidDay;
  if (peaceD >= 20) streakHTML += ` · <span title="Days since last raid" style="color:#9bcfa9">☮️${peaceD}d</span>`;
}
if (G.stats?.citizensDied >= 1 && G.lastDeathDay !== undefined) {
  const lifeD = G.day - G.lastDeathDay;
  if (lifeD >= 30) streakHTML += ` · <span title="Days since last death" style="color:#bdb09a">🕯️${lifeD}d</span>`;
}
```

The `${streakHTML}` interpolation appends to
the day-display string between `${raidWarn}`
and the difficulty label.

15 LoC change (10 LoC code + 5 LoC comment).

## why this shape

- **Threshold-gated visibility.** Peace
  shows at 20+ days; life at 30+ days. Below
  those, the indicator hides — avoids HUD
  flicker as raids happen frequently in
  early game.
- **Earned-state gate matches the beats.**
  `raidsSurvived >= 1` and `citizensDied >= 1`
  ensure brand-new realms don't see either
  indicator. Mirrors the gates 211/228 use
  for their beats.
- **Mutually-exclusive with raid warning.**
  Peace indicator only renders when NO raid
  is imminent (`!raidDays`). When a raid is
  3 days out, the player sees ⚔️ countdown,
  not the now-stale peace count.
- **Distinct icons + colors:**
  - ☮️ peace — soft sage green (#9bcfa9)
  - 🕯️ memorial life-streak — warm tan
    (#bdb09a)
  Both desaturated so they don't compete
  with primary HUD information.
- **Photo-mode hidden** via the existing
  `body.photo-mode #hud` rule (no extra CSS
  needed).
- **No new state**, no save changes — uses
  the existing 211 + 228 trackers.

## what I did NOT do

- **Full-pop indicator.** 230's
  `lastUnderpopDay` is also tracked, but
  population vs maxPop is volatile in a
  growing realm — adding a "X days at full
  pop" indicator would flicker. Filed for
  later if a stable threshold (e.g.,
  pop>=10 + 30+ days) reads well.
- **Per-streak prose tooltip.** Currently
  the title attribute is just "Days since
  last raid"; could be more evocative
  ("the bell has hung silent for X
  evenings"). Defer.
- **Animate the icons** when streaks pass
  notable thresholds (e.g., 50/100 days).
  Filed.
- **Add to chronicle when streaks tick over
  notable boundaries.** Already covered by
  the 211/228 beats (which fire at 50/100
  days respectively).
- **Mobile responsive layout.** The day-
  display already truncates on narrow
  viewports per 041's CSS work; the new
  spans inherit that.
