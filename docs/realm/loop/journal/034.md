## the fix

**Three new ensure functions** in `js/story.js`,
modeled exactly on `ensureMayor/ensureBard/ensureRival`:

- `ensureSmith()` — picks from SMITH_NAMES, hooks into
  `G.namedCharacters.smith`, fires a chronicle line.
- `ensureMerchant()` — same pattern for markets.
- `ensureTeacher()` — same for schools.

Each has a name pool of 6–7 evocative-medieval names
(e.g. "Harald Ironhand", "Osric the Coin", "Dame Ysolt
the Wise") and a first-arrival chronicle line:

- Smith: *"{name} takes up the hammer at the new forge.
  'Steel remembers every strike,' they say."*
- Merchant: *"{name} sets up stall at the market.
  Their caravans travel as far as the coast."*
- Teacher: *"{name} opens the schoolhouse doors. The
  first lesson: 'Know thy realm.'"*

**Extended `BUILDING_FIRST_BEATS` (026's data table)**
with an optional `after` field. Entries for
`blacksmith`/`market`/`school` now carry
`after: ensureSmith` / `ensureMerchant` /
`ensureTeacher`. Loop body runs `beat.after()` after
`chronicle(beat.text, 'milestone')` when present.

This extends 026's "pure data" aesthetic slightly —
function references in data, which 026 called
"fragile." For character-setup side-effects it's
tolerable: the ensure functions are idempotent
(re-call short-circuits if already set), and the
alternative (moving these 3 beats back inline)
would duplicate 026's consolidation work for no
structural gain.

## what this unlocks

The `G.namedCharacters` map now has up to 6 entries
(mayor, bard, rival, smith, merchant, teacher). Other
code that references named characters can now cite
them in richer contexts — e.g., the castle-built
chronicle line already interpolates the mayor's name;
similar moments could reference the smith when a
sword is forged, the teacher when a research
completes, etc. Out of scope for this tick, but the
infrastructure is now there.

## what I did NOT do

- **Wire more NPCs.** The pattern supports adding
  (e.g., fisher for Fisherman's Hut, baker for
  Bakery). But the 3 021 named explicitly are
  enough for now; scope creep is exactly what
  MANIFEST's "one challenge per tick" prevents.
- **Cross-reference named characters in later
  chronicle beats.** Noted as potential future work
  above, not shipped.
- **Refactor the `after` field out.** 026's "fragile"
  concern was legitimate; if a future tick finds the
  mixed-data problem biting, they can revert to
  inline beats for the 3 side-effect rows.
