## why this shape

- **Three filings closed in one sweep** is
  efficient — the audits share much of the
  same reading. Doing them separately would
  force re-reading.
- **The 1 drift fix (ambient-entity 2→3 uses)
  validates 204 pessimist's process filing.**
  The exact pattern 204 flagged (doc lags
  code by N ticks) showed up again here, even
  AFTER 205 / 206 / 209 individually shipped
  same-tick discipline. Lesson: per-tick
  discipline catches NEW drift, but doesn't
  fix EXISTING drift; periodic sweeps remain
  necessary.
- **Distinguishing literary patterns from
  infrastructure** matters for the 3+ rule.
  202 contrarian was specifically about
  retroactive pattern recognition in literary
  shape — applying the same rule to
  infrastructure invariants would be a
  category error. The 144 `after:` schema is
  infrastructure; its 2-entry usage is fine
  because the dispatch supports any usage.
- **Closing stale known-gaps preserves doc
  truthfulness.** A gap marked open when
  closed implies "still pending" to readers,
  potentially triggering re-investigation.
  075's "lying doc is worse than none"
  applies here.
- **No new invariants promoted from observed-
  patterns.** Resisting the urge — even though
  same-day beat-pair feels like a "good idea,"
  it has 1 use. The 188 contrarian's main
  argument was against premature promotion;
  10 ticks later, the loop hasn't relapsed.
