## the build

`docs/realm/js/render.js:2584` —
1-character fix to particle cull check:
include `p.offsetX` in the psx
calculation.

```diff
- const psx = (s.x - G.camera.x) * G.camera.zoom + logicalW / 2;
+ const psx = (s.x + (p.offsetX || 0) - G.camera.x) * G.camera.zoom + logicalW / 2;
```

Plus a 5-line comment citing 273 [code]
rationale.

## why this matters

The cull check at line 2584 decides
whether a particle gets rendered. Pre-274,
it computed psx using the BASE TILE
position only (`s.x`), ignoring any
offsetX. Particles with large offsetX
(267 shootingstar accumulates up to
±200; 273 lightning uses +120) could be:

- **Falsely culled**: base tile in
  viewport, offset position out of
  viewport — particle drawn off-screen,
  cull check passes, wasted work.
- **Falsely uncullled**: base tile out of
  viewport, offset position in viewport —
  particle drawn in viewport but cull
  thinks it's outside, particle skipped.
  This is the dangerous case.

273 caught the FALSELY-UNCULLED case
incidentally while fixing 267's spawn
math. With both 267 + 273 spawn-math now
correct (using inverse iso to put base
at camera focus), the cull check
becomes more important — large-offsetX
particles like the lightning at +120
SHOULD pass cull when their actual
position is on-screen.

The cull tolerance (±50) made small-
offsetX particles work, but offsetX up
to 200 (267 streak end) plus zoom
1.3+ multiplier could push the
EFFECTIVE drawn position 260+ pixels
outside the cull bounds while base
tile says it's inside.

This 274 fix makes the cull check
ACCURATE rather than approximate.
