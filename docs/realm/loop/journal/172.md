## why this shape

- **Strategic plan IN the doc, not just the
  journal.** 171's journal contains the full
  reasoning; the doc contains the actionable plan.
  Future fresh-context ticks read the doc, not the
  journal. Codifying the plan there means it
  survives context loss.
- **Phase structure preserved, dates estimates.**
  The 3-phase scaffolding (sprite-roster
  completion → integration → animation) is the
  durable artifact. Specific tick numbers (180,
  185, 188) are pace projections, not contracts.
- **`.glb` boundary made EXPLICIT.** The user has
  let `.glb` files change in their `git status`
  for ~170 ticks without ever asking the loop to
  touch them. 172 makes the boundary durable so
  no future tick wastes effort there.
- **Coverage-map header date updated.** Small
  hygiene; the data was current but the date
  said 165. Stale date markers erode trust in
  the doc.
- **No code change.** Pure doc-update tick. The
  171 plan is now anchored.
