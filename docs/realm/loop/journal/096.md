## the change

`docs/realm/js/render.js:_WINTER_CAPS` — 3 y-offset values:

```js
lumber:   { w: 11, h: 3, y: -28 },  // was -29 (floated above chimney)
windmill: { w:  9, h: 3, y: -28 },  // was -32 (floated above sails)
granary:  { w: 15, h: 3, y: -22 },  // was -26, h=4 (stacked-dome look)
```

Also changed granary's `h: 4 → 3` to match the baseline
height — the extra-thick cap wasn't perceptually doing
anything once the y-offset was lowered to hug the dome.

## why this shape

- **Width variation preserved.** The three buildings still
  have w=11 (lumber), w=9 (windmill), w=15 (granary) vs
  baseline w=13 (house). Pixel-diff silhouette audits still
  see unique widths per-type.
- **yOffset axis de-prioritized.** 094 treated yOffset as
  an equal axis to width; 095 showed yOffset changes
  perceptually read as "cap is floating" when pushed above
  a building's actual roofline. 096 converges y-offsets
  toward a safe baseline (-28) while letting width do the
  disambiguation work.
- **Graceful "default" still exists.** Any new building
  type not in the table gets `_default: { w:14, h:3, y:-28 }`.
- **Preserves icicles.** Scale with `snowW` — narrower caps
  still generate narrower icicle strips; that natural
  variation sticks.
