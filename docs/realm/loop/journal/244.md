## the build

Two changes in `js/story.js`.

### 1. New BUILDING_FIRST_BEATS entry

```js
{ type: 'townhall', flag: 'firstTownHall',
  text: G => {
    const m = G.namedCharacters?.mayor;
    return m
      ? `The town hall opens. ${m.name} sits at the long table for the first time; the door is locked at dusk for the first time. The realm has minutes that will outlast it.`
      : 'The town hall opens. The door is locked at dusk for the first time. The realm has minutes that will outlast it.';
  } }
```

Civic-formal register. Contrasts with the
sensory openings of tavern (laughter), church
(bells), barracks (drill at dawn). Townhall's
opening is procedural — sitting at a table,
locking a door. Closes with "the realm has
minutes that will outlast it" — bureaucracy
as a form of immortality, gentle joke at
governance's procedural nature.

Mayor reference closes the loop between
243's structural-unlock (mayor required to
build) and the chronicle (mayor as the
person at the table).

Defensive fallback when mayor somehow not
named — should never trigger in practice
because 243's gate requires mayor, but the
fallback prevents `${undefined.name}` if a
load-order edge case occurs.

### 2. Dispatch upgrade (same-tick fix)

`BUILDING_FIRST_BEATS` dispatch at line
~883 now handles function-text:

```js
const text = typeof beat.text === 'function' ? beat.text(G) : beat.text;
chronicle(text, 'milestone');
```

Mirrors the NARRATIVE_BEATS dispatch pattern.
Backward-compatible with all existing static-
text entries (tavern/barracks/church + the
13 BUILDING_FIRST_BEATS data-driven entries
in the table above).
