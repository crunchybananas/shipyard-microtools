## why this shape

- **Combine into one tick.** Both changes
  touch `_VARIANT_PALETTES` semantics; both
  share `verify-variants.mjs` as the test
  surface. Splitting would double overhead
  for no benefit. (Per 257 [process] rule:
  "single doc/file target AND each ≤30
  LoC, combine.") Verified path holds.
- **Text-level assertion, not pixel
  comparison.** Pixel comparison would be
  fragile (anti-aliasing, day/night tint,
  zoom). Text-level swap is the production
  pipeline's truth — assert that, not
  rendering artifacts.
- **`replaceAll` order independence.**
  Test asserts target color presence AND
  source color absence — catches incomplete
  swaps (e.g., if target color happens to
  be a substring of source).
- **Hash-distribution check.** A 4-kname
  test that always lands on idx=0 would
  spuriously pass other assertions (no
  swap = no failure). The "≥2 distinct
  indexes" assertion guards against
  degenerate hash distributions.
- **Townhall variants don't swap accent
  colors.** Pediment / windows / cupola
  stay constant. Reasoning: too many
  swappable colors fragments the silhouette
  recognition. Façade swap is enough to
  read as "different stone" while keeping
  "this is a townhall" instantly legible.
