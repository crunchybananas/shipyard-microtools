## what house.svg expresses that canvas struggles with

- **Half-timber wall accents.** Canvas drawHouse
  has flat rectangle walls; real medieval houses
  often had visible timber framing. SVG ships
  3 vertical posts + 1 diagonal cross-brace at
  ~38% alpha black-brown — gives Tudor character.
- **Smoke wisps from chimney.** Canvas chimney
  is a simple painted rect; SVG adds 3
  successively smaller `<ellipse>` smoke wisps
  drifting up-and-left toward the sun. Reads as
  occupied/inhabited.
- **4-pane mullion cross on window.** Canvas
  window is a single `fillRect(s.x+4, s.y-18, 3,
  3)` with `'#ffeebb'` fill. SVG ships frame +
  light pane + horizontal+vertical mullions
  dividing into 4 panes + corner highlight
  reflection.
- **Flowerbox below window.** Canvas has no
  flowerbox; SVG adds a small wooden box with
  3 colored flower-dots. Reads as
  "lived-in / cared for."
- **Iron studs + ring handle on door.** Canvas
  door is `fillRect(s.x-3, s.y-10, 6, 6)` —
  plain dark rect. SVG ships planks + frame +
  iron studs + ring handle.
- **Ridge tiles + tile rows on roof.** Canvas
  roof is a 3-vertex triangle fill; SVG adds 5
  horizontal tile-row lines suggesting overlapping
  clay tiles + a darker ridge tile bump.
- **Plaster texture pattern.** Subtle horizontal
  weather lines via `<pattern>` overlay.

## what I changed in form vs canvas

- **Variant fixing.** Canvas picks per-house
  hash; SVG ships ONE canonical variant. This
  matches my earlier decision on church.svg
  (canvas: 4 glass palettes × 4 ornaments;
  SVG: classic blue glass + bell). Phase B
  integration step 5 handles variation.
- **Chimney always right-side.** Canvas
  alternates left/right; SVG always right.
- **Always single window.** Canvas may add
  extra window (~50% via hash bit); SVG always
  one. Could add a 2-window variant SVG file
  during Phase B.
