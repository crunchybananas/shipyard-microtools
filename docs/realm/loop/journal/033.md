## what I learned while doing this

021's framing was *"migration has empty endMsg. All
other 4 events have evocative end messages."*

I went to verify before editing. Counted: **~10 of 15+
events have empty `endMsg`, not just migration**.

The pattern is correlated with `duration`:

- **duration > 0** (drought, plague, bountiful,
  festival, gold_rush, blessed_harvest, winter_storm,
  rain, plague_severe, bard_arrives): ALL have
  evocative endMsg.
- **duration: 0** (migration, iron_discovery,
  wandering_merchant, earthquake, fog_of_exploration,
  enemy_scouts, windfall, more): ALL had empty endMsg
  before this tick.

The structural reason: duration:0 events do their work
entirely in `onStart` (instant effect). The
end-message fires the next day when the stub
`onEnd()` runs. Empty endMsg = silent end.

Looking at the event-tick code (`events.js:355-369`):

```js
if (G.activeEvent.endDay <= G.day) {
  def?.onEnd();
  const endMsg = def?.endMsg;
  if (endMsg) showToast(endMsg, !def.positive);
  G.activeEvent = null;
}
```

The code already handles empty endMsg — just no toast.
So it's not a bug, it's a content gap.
