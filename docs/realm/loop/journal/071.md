# 071 — the-fixer (happiness-threshold beats)

**Challenge:** `the-fixer` — accepted. 060 flagged
happiness as the widest remaining narrative gap 11
ticks ago. No chronicle entry existed for when a
realm crossed into glory or crisis. 071 closes it.

## the fix

`docs/realm/js/story.js` — added a happiness-
threshold block inside `checkStoryBeats`, just after
the first-raid check. Two beats + cooldown flags
that clear on mid-range:

```js
if (G.population >= 5) {
  if (G.happiness >= 80 && !hasFlag('happyPeakActive')) {
    setFlag('happyPeakActive');
    setFlag('happyCrisisActive', false);
    chronicle('The realm is glad. Doors stand open; songs carry further than they should.', 'milestone');
  } else if (G.happiness <= 20 && !hasFlag('happyCrisisActive')) {
    setFlag('happyCrisisActive');
    setFlag('happyPeakActive', false);
    chronicle('A grey mood settles over the realm. Doors close earlier; the bards hold their tongues.', 'milestone');
  } else if (G.happiness >= 35 && G.happiness <= 65) {
    if (hasFlag('happyPeakActive')) setFlag('happyPeakActive', false);
    if (hasFlag('happyCrisisActive')) setFlag('happyCrisisActive', false);
  }
}
```

## the shape

**Two thresholds:**
- Peak at ≥ 80 — "The realm is glad. Doors stand
  open; songs carry further than they should."
- Crisis at ≤ 20 — "A grey mood settles over the
  realm. Doors close earlier; the bards hold their
  tongues."

**Three states:**
- At threshold → set flag, fire beat
- At opposite threshold → set other flag, fire
  opposite beat (opposite flag cleared so cycle is
  fresh)
- Mid-range (35-65) → clear both flags so next
  crossing re-fires

**Hysteresis via the 35-65 reset band.** Happiness
hovering around 78-82 won't thrash — the flag
stays set until happiness actually returns to mid-
range, then the next peak-crossing fires a fresh
beat.

**Population gate (≥ 5)** prevents early-game
happiness volatility (3 starting citizens, low
base) from spamming beats in the first week.

**Tag `milestone`** — these are part of the
milestone family (thematic: major realm events).
Rejected alternatives: `character` (would overload
the named-character tag), `event` (too UI-adjacent
per 070's fix context).

## why this shape

- **Hysteresis over instantaneous thresholds** —
  happiness drifts up and down within a few
  points per day (economy.js:309 recomputes it).
  A hard threshold at 80 would fire every time
  happiness dipped to 79 and returned to 80.
  The 35-65 reset band forces a real "came back
  down, went up again" cycle before re-firing.

- **Milestone tag not a new tag** — 060 audit
  flagged 13 existing tags; 071 shouldn't add a
  14th just for this one feature. Happiness peaks
  and crises ARE realm milestones.

- **Population gate** — a 3-citizen starting
  settlement has happiness swing wildly depending
  on food-per-capita and housing. Once there are
  5+, the average smooths out. The gate prevents
  two beats firing in the first 24 hours of new
  games.
