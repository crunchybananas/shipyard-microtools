## the gallery (5 frames)

### frame 1 — winter midday (dayPhase=0.5)

Clean read. Castle anchors upper-right with red peaked roofs +
yellow flag. Village clusters around it with visible
variety: snow-dusted farms (087's rhombus + frost pixels), tavern
with 051 flag above cap, windmill with tight cap (096 fix), dome
granary with cap hugging the dome (096 fix). Bright white snow
+ pale blue sky. Reads as a winter settlement.

**Verdict:** readable, honest, but flat. Midday winter lacks
atmosphere.

### frame 2 — winter dusk (dayPhase=0.78) ⭐ COVER-ART

Deep plum-rose sky with visible moon upper-right. Ground shifts
to warm amber-purple via 012's dusk hue-lerp. **Castle windows
glow golden** against the dusk — the night-window-glow code
(render.js:2996) kicks in at daylight<0.75. Several other
buildings (tavern, blacksmith, windmill) also glow. Snow + glow
creates warm-cold contrast that sells "inhabited realm" mood.

**Verdict:** the cover-art frame. Everything composes:
temperature contrast, light hierarchy, readable silhouettes
(094/096 graphics), focal castle, atmospheric sky. If realm
needs a marketing screenshot, this is the recipe.

### frame 3 — winter deep night (dayPhase=0.05)

Pitch-black sky. Only the glowing windows give hierarchy; the
snowy ground barely visible. Very atmospheric but also very dark
— could be argued as the ugliest frame (little information) or
the most mood-heavy depending on taste.

**Verdict:** atmospheric but dark. Not suitable for cover-art
(buyer needs to see WHAT the game is). Works as a rare
"it's lovely at night" side-show.

### frame 4 — summer dusk (dayPhase=0.70)

Warm amber-tan ground (017's summer tileShift), purple-rose sky
gradient, green farm crops visible (no winter dust). Castle
glow still present. The 018-recipe's classic golden-hour frame.

**Verdict:** strong runner-up to winter-dusk. If the game wants
a "harvest" vs "winter" cover-art diptych, this pairs cleanly
with frame 2.

### frame 5 — winter dawn (dayPhase=0.22)

Rose-salmon ground tint, bright rising sun upper-left, deep
purple fading to rose sky gradient. Castle + tavern glow
still active at dawn. **Surprise finding below.**

**Verdict:** honest morning frame; second-best
cover-art candidate for winter if frame 2 isn't available.

## surprise finding: green crops in winter farms

Frame 5 (and looking back, frames 1 and 2) clearly show farms
with GREEN CROPS visible underneath 087's snow-dusting overlay.
087 uses 0.35 alpha on the rhombus — intentionally transparent
so the tile reads underneath — but the farm sprite's crop
colors are painted as full-summer green and they show through
the thin white dust.

**File:** `render.js:2981` (087's rgba(230,240,255,0.35)
semi-transparent rhombus). At 0.35 alpha the underlying green
is still perceptually GREEN in winter, breaking seasonal
consistency.

**Options to fix:**
1. Raise farm/quarry dust alpha to 0.55-0.65 (sacrifices some
   sprite readability, gains seasonal consistency)
2. Add a second overlay: a desaturation layer that grays the
   crops without fully covering them (more complex)
3. Paint winter-variant farm sprite with dormant brown crops
   (biggest change, most "right" answer, but touches drawFarm
   which has many existing knobs)

Option 1 is the 094-style minimum-viable fix. Option 3 is the
correct one if winter-farm gets its own tick's attention.

## secondary finding: building labels in photo-mode

Frames 1, 2, 3, 5 all show "0/1" worker labels above buildings
in photo-mode. 035 said "body.photo-mode CSS hides ~15 HUD
elements" but per-building worker counts render as WORLD-layer
UI, not HUD — they're drawn on the canvas by the game's rendering
code. Photo-mode doesn't hide them.

**File:** per-building worker labels render somewhere in
render.js (likely the building UI-layer loop). Adding a
`G.photoMode ?` gate around that block would clean up photo frames.

## what the photographer had to do

To get frame 2's cover-art shot:
1. Build a cluster with a castle focal point (not just tower)
2. Force `G.season = 'winter'`, mid-winter day (22-28 OK)
3. Force `G.dayPhase = G.dayLength * 0.78` (dusk peak)
4. Toggle photo-mode (H key / `togglePhotoMode()`)
5. Zoom ~1.8, center camera on cluster (use iso math to
   convert tile→camera coords)
6. Wait for at least one `simTick` so window-glow + particle
   state have updated
7. Screenshot

For a live player: steps 1, 4, 5, 7 are natural; steps 2, 3
require waiting for actual dusk in winter (realistic) OR
using the console (`G.dayPhase = 3600 * 0.78; G.season =
'winter'`). A "photo preset" helper would be ideal — 018 filed
this idea ("photo-mode preset: codify t=0.70 + summer as the
cover-art recipe when photo-mode ships") and it's still open.
