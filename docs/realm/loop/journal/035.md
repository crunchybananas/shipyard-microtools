## what shipped

A minimum-viable photo mode, triggered by the `H` key.

**Scope summary**:
- New flag `G.photoMode` in `state.js` (default false)
- `window.togglePhotoMode()` in `main.js` — flips flag
  and toggles `body.photo-mode` class
- `H` key binding in `input.js` (next to the existing
  m/l/r shortcuts)
- CSS `body.photo-mode` rule in `index.html` hiding
  ~15 HUD elements (`#hud #build-bar #missions
  #controls #minimap #info-panel #tutorial-tip #toast
  #activity-feed #event-banner #chronicle-panel
  #happiness-panel #research-panel #stats-panel
  #achievements-panel #notification-log #trade-panel
  #pop-panel`)
- `renderPauseOverlay` in `enhancements.js` early-
  returns when `G.photoMode` — hides the "⏸ PAUSED"
  label so you can screenshot a paused frame clean
- Small `#photo-mode-badge` at bottom-center: *"📷
  Photo Mode — press H to exit"* — only visible when
  photo mode is on; bottom position stays out of
  cover-art frames

**Net: 5 files touched, ~30 lines added.**

## what I did NOT do

- **Photo-mode preset** (018 idea). The recipe is
  "set dayPhase=0.70, season=summer, pause." Could
  ship as a second keybinding (e.g. `Shift+H` =
  photo-mode + sunset preset). Left for a later tick;
  the `H` toggle alone is the atomic MVP.
- **Save the photo mode to local storage.** If you
  reload while in photo mode, you're back to normal
  HUD. Considered trivially small but out of scope —
  probably the right default (photo mode is a
  moment, not a session).
- **Hide the speed / cinematic-zoom on-tick
  animation.** The game still runs background tick
  animations (day/night drift, wandering birds,
  citizen walk cycles). That's correct — photo mode
  is a HUD-hide, not a world-freeze. Pause separately.
- **Scope-extend to other chrome.** The `#toast`
  element is hidden, as is the `#activity-feed`. If
  a future tick finds something else bleeding through,
  add its selector to the CSS rule.

## ideas generated → ideas.md

Two noted but not filed (the 018 preset idea is
already on the list):

- Minor: expose `G.photoMode` via URL query
  (`?photo=1`) for marketing-page direct-link-to-
  cover-art scenarios. Not filed — feels feature-
  y rather than necessary.
- Minor: photo-mode badge should fade out after 3
  seconds rather than persist. Current behavior:
  badge stays while photo mode is on. The case for
  persistence: it's the *only* hint that `H` exits.
  The case for fade: cleaner screenshots. I left it
  persistent because a screenshot would just mask the
  badge by cropping it out anyway.
