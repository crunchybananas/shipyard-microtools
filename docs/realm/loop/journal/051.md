## the fix

Nine lines inside `drawTavern` (render.js:4197), right
after the per-tavern `signColors` palette is picked
(27's work from the original side-sign variant).

```js
// Projecting roof-peak flag — pole + flag above y-30 peak
ctx.fillStyle = '#3a2818';
ctx.fillRect(s.x - 0.4, s.y - 36, 0.8, 6);     // pole
ctx.fillStyle = signColors.board;
ctx.fillRect(s.x + 0.4, s.y - 36, 5, 2.5);     // flag
ctx.fillStyle = signColors.accent;
ctx.fillRect(s.x + 0.4, s.y - 36, 5, 0.5);     // top band
ctx.strokeStyle = 'rgba(0,0,0,0.4)';
ctx.lineWidth = 0.4;
ctx.strokeRect(s.x + 0.4, s.y - 36, 5, 2.5);   // outline
```

Geometry:
- Roof peak is at `y-30`. Pole rises from `y-36` to
  `y-30`, offset ±0.4 px from the building centerline
- Flag extends 5 px to the RIGHT of the pole,
  `y-36` to `y-33.5`
- Accent band is 0.5 px along the flag's top edge
  (matches the side-sign's top-band convention from
  27)
- 0.4px-wide dark outline so it reads against bright
  sky

Reuses `signColors.board` + `signColors.accent` — the
per-tavern palette already varied for the side-sign
(gold / burgundy / navy / moss). Same tavern's flag
and side-sign now share colors, so a player who sees
both reads them as "the Boar's Head" vs "the Anchor"
visually.

## why this breaks the 046 blur

046 found:

> house ≈ tavern ≈ bakery share peak-roof + chimney
> silhouette at default zoom. Chimney width deltas
> are 2-3 px on a 20×28 bbox — effectively
> indistinguishable in pure black silhouette.

The side-sign from 27 sits at `y-16..y-8` — below the
roof line. In a silhouette rendering (any dusk frame,
any minimap, any accessibility high-contrast mode),
it's part of the wall rectangle, not a distinguishing
feature.

The new peak-flag adds a **horizontal bar above the
roof triangle**, at the same elevation as the
house's chimney. Now:

```
HOUSE            BAKERY           TAVERN
  .|.              .|.             .___
 .| |.            .| |.           ./   \_
 /   \            /   \           /    |.
/_ _ _\          /_ _ _\         /_ _ _ \
|  o  |          |  o  |         |sign  |
|_____|          |_____|         |______|

chimney-         chimney-         FLAG
above-peak       above-peak       (horizontal,
(vertical)       (vertical,       side of
                  thicker)        centerline)
```

The flag projects RIGHT of the pole, giving tavern
a distinct ASYMMETRIC top-element where house and
bakery have CENTERED verticals. Silhouette
differentiation is now a topological difference
(shape family), not a width difference (2-3 px).

## what I did NOT do

- **Animate the flag waving.** Tempted. But 048's
  unnamed-thing is currently the ONLY animated
  ambient element, and 050's letter noted the
  windmill blades are the only other shape-changing
  silhouette in the game. Keeping the flag static
  preserves 048/windmill's uniqueness.
- **Project the sign LEFT instead of right.** 27's
  side-sign projects LEFT from the wall. Projecting
  the roof-flag left too would emphasize the
  building's left side. Right feels more balanced
  in isometric view where the camera angle already
  shows the right-face.
- **Apply the same fix to bakery and house.** That
  would be closer to "re-skin the residential
  cluster." The goal was to BREAK the cluster, not
  harmonize it. Bakery's oven-bulge idea from 046
  and the house's chimney-variant system (loop 18)
  keep those two in their own identity lanes.
- **Tune the flag colors for night visibility.**
  The existing signColors go through the same
  night-tint multiply that every other sprite does
  (render.js:2515+). The 012 hue-lerp + the per-
  tavern variants should handle this organically.
- **Actually pixel-verify on canvas.** Budget
  constraint + 046's tooling gap.
