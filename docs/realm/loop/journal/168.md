## what windmill.svg expresses that canvas struggles with

- **Cloth-paneled sails.** Canvas drawWindmill
  has BLADES (line 5295-5301: `fillRect(-1.5,
  -12, 3, 12)`) — solid wooden rectangles.
  Real windmill sails have FRAMES with cloth
  panels stretched across them. SVG ships all
  6 cloth panels per sail × 4 sails = 24
  cloth rects + supporting frame structure,
  with cloth-seam linework. Canvas would
  require ~50 lines per sail × 4 = 200+ lines
  of imperative drawing.
- **Stone-block pattern overlay on tapered
  trapezoid.** Canvas does horizontal mortar
  lines via for-loop on a flat rectangle. SVG
  applies `fill="url(#windmillBlocks)"` to
  the actual trapezoid path — pattern follows
  the building's silhouette correctly.
- **Iron hub with radial gradient + bolt
  detail.** Canvas hub is a single
  `ctx.arc(0, 0, 3, 0, 2π)` with `'#5a3a18'`.
  SVG hub layers: outer iron radial-gradient
  ring + inner dark pupil + 4 bolt highlights
  + specular dot. ~7 elements vs 1 canvas
  call.
- **Wooden cross-frame inside each sail.**
  Visible wooden spar + 2 arms structure that
  the canvas omits entirely — canvas blades
  are visually solid panels.
- **Crisp at photo-mode zoom.** Cloth-seam
  lines (0.2 stroke-width), bolt highlights
  (0.4 r), door hinges (0.5 height) all stay
  sharp at 5× SVG zoom.

## what I changed in form vs canvas

- Canvas blades **rotate** (animated via
  `Math.sin(G.gameTick * 0.03)`); SVG sails
  are static at canonical cross pose. Matches
  167 3D mesh (also static). Animation
  deferred to a future `<animate>`-based tick.
- Canvas has 4 SOLID blades; SVG has 4
  CLOTH-PANELED sails with frames — visually
  richer, more authentic to historical
  windmill construction.
- Canvas has a **trapezoid stone base**
  (wider at bottom); SVG matches this with
  `<path d="M 50 102 L 78 102 L 74 80 L 54
  80 Z">` — same shape, declarative.
