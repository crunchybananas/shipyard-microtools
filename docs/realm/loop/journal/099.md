## the changes

`docs/realm/js/render.js` — two edits:

### fix 1: winter-dust alpha 0.35 → 0.55 (render.js:2981)

```js
ctx.fillStyle = 'rgba(230,240,255,0.55)';  // was 0.35
```

Plus the frost-highlight pair:
```js
ctx.fillStyle = 'rgba(255,255,255,0.55)';  // was 0.35
```

Raises the winter overlay's opacity on farm + quarry tiles.
0.55 preserves some sprite detail underneath (buyer can still
tell "this is a farm, not a blank tile") but makes "this is
snowy" the dominant perceptual signal.

### fix 2: photo-mode gate on worker-count labels (render.js:3119)

```js
if (needed > 0 && !G.photoMode) {   // was: if (needed > 0)
```

Prevents the "0/1👤" label from rendering when photo-mode is
active. 035's CSS-based photo-mode only hides DOM elements;
canvas-drawn UI requires code-level gating.

## why this shape

- **Both 098 findings are graphics-layer bugs,** one canvas
  overlay and one canvas text. Pairing them in one tick is
  natural (both in render.js, both discovered in same play).
- **099-alpha=0.55** was a range 0.55-0.65 in 098's filed
  fix. Picked 0.55 (low end) because: (a) 094/096 showed
  that perceptual changes compound surprisingly; minimum
  viable change; (b) 0.55 still lets the tile outline
  read, so the farm doesn't look like a blank snow patch.
- **Photo-mode gate on labels,** not removal of labels.
  Players OUTSIDE photo-mode still need the info ("how
  many workers is this building missing?"). Gating is the
  non-destructive fix.
- **No new photo-mode check infrastructure.** `G.photoMode`
  is already a globally-checked boolean (035 shipped it).
  Just add `!G.photoMode` where needed. Matches the existing
  pattern.
