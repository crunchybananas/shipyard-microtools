## the build

Three concrete deliverables open the SVG axis:

### 1. `docs/realm/assets/sprites/granary.svg`

A richly-detailed SVG sprite of the granary,
sibling to 158's canvas `drawGranary`. Same
building, different render approach. ~110 lines of
SVG. Demonstrates what's now affordable to author:

- **Gradients** (`linearGradient` for silo body,
  `radialGradient` for dome face + grain pile +
  ground shadow). Smoother light falloff than
  canvas `createLinearGradient` chains and stays
  crisp at any zoom.
- **Pattern fill** (`pattern` for wood-grain on
  dome). Tileable, vector-clean, would be tedious
  in canvas.
- **Per-shape highlight + shadow strokes** that
  don't pixel-stair under photo-mode zoom (5×).
- **Decorative detail at the same authoring cost**:
  finial knob, hinges, latch, plank seams, hoop
  studs, grain specks, dust flecks. Each detail
  costs ~1 line of SVG markup vs ~5+ lines of
  canvas instructions.
- **Light direction**: upper-left sun, matching the
  canvas pipeline's implicit convention.
- **Accessibility**: scaled the viewBox to 128×128
  so the sprite renders crisp from 48px (default
  game zoom) to 384px (photo-mode max).

### 2. `docs/realm/svg-test/index.html`

A sandbox page that loads `granary.svg` at four
sizes (48 / 96 / 192 / 384 px) on a grass-toned
gradient stage, with a meta panel listing what the
SVG does that canvas can't easily, and open
questions for future ticks. Standalone — no JS, no
build step. Open via `python3 -m http.server` from
the realm root.

### 3. Memory + index entries

`memory/project_realm_loop_render_axis.md` captures
the steering as durable user preference. MEMORY.md
indexed.

## why this shape

- **Doesn't break anything.** No existing
  `drawGranary` change. The canvas version still
  ships; the SVG is a parallel sandbox.
  Side-by-side coexistence lets future ticks
  compare without committing to a migration.
- **Sandbox before integration.** Replacing
  `drawGranary` in the live game with `<img
  src=granary.svg>` would touch the render
  pipeline (transforms, day-night tint, winter
  cap, fog, hover tooltips). One sandbox tick
  first; integration tick later if the
  experiment lands.
- **Concrete deliverable, not a survey.** User
  said "really focus on" — that's "ship
  something," not "write a plan." The SVG is the
  ship.
- **Pairs with 158.** 158 polished the granary's
  canvas-side dome cap; 161 demonstrates the SVG
  alternative to the same building. Two ticks,
  same subject, different render layers.
- **Granary specifically.** Out of 11 buildings,
  granary has the most varied form (cylinder +
  dome + sacks + grain pile + arched door + hoops)
  — best showcase for what SVG can express that
  canvas struggles with.
- **No 3D this tick.** The 3D prototype
  (`docs/realm/3d/3d.js`) is genuinely
  parallel — opening that axis deserves its own
  tick rather than splitting attention.
