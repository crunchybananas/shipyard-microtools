### 1. Kingdom-hashed procedural asterism

```js
function _kingdomConstellationPts() {
  // Cached per kingdom. Hash "${kname}_constellation_pts"
  // with the same char-fold family as story.js:_dreamHash.
  // nPts = 5 + (Math.abs(h) % 3)  → 5, 6, or 7 points
  // fx ∈ [0.15, 0.85], fy ∈ [0.05, 0.22]  (upper sky band)
}
```

Per-kingdom deterministic. 5-7 star points placed in
the upper 17% vertical band, spread across middle
70% horizontally. Cached — hashed once per kingdom,
reused every frame.

### 2. Gated render

```js
if (!G.storyFlags?.constellation_named) return;
```

The asterism only appears AFTER 116's beat has fired
— the realm has to NAME the constellation before it
appears in the sky. Young realms see an unmarked
sky. This gives 116's beat a quiet visual payoff
(autumn days later, players notice the stars they
named are actually there).

### 3. Rendering

Faint connecting lines (alpha 0.15 × nightStrength)
linking the stars in index order. Brighter vertex
stars (alpha 0.9, radius 1.7px) set against the
default 180-star field. nightStrength ramps 0→1 as
daylight 0.7→0.45 — asterism fades in during dusk,
holds through night, fades out at dawn.

Removed the per-vertex cross sparkle from 016 — the
180-star field already handles twinkle, and adding
crosses only on named stars drew too much attention.
The vertex stars are now just slightly brighter
circles; the shape is read by eye.

## why this shape

- **Procedural, not per-shape-name curated.** 116's
  20-name pool (the Hare / Ash-Tree / Broken Wheel
  / ...) is evocative but curating 20 literal
  patterns would have eaten 200+ lines of point
  data. Real asterisms don't look like their
  names either ("the big dipper" is 7 dots; the
  "dipper" is a pattern-recognition projection).
  Procedural hashing preserves that quality: each
  kingdom has a distinct shape, but the shape is
  abstract until the name frames it.
- **Gated on `constellation_named`.** Un-named
  realms see a blank sky. The beat earns the
  visual.
- **Cached by kingdom.** `_constShapeCache` keyed
  on kingdom name. Save/load safe — no persisted
  state (recomputes on first frame post-load).
  Kingdom-name change (via hypothetical future
  rename) would regenerate automatically.
- **Faint lines, brighter vertices.** 016's
  original alpha of 0.18 / 0.95 was retained on
  vertices (slightly bumped to 0.9) but the
  connecting lines dropped from 0.18 to 0.15 to
  sit under the 180-star field rather than
  competing with it.
- **Upper-sky band only (fy ≤ 0.22).** Stars
  below 22% of screen height would overlap the
  game world. Keeps the constellation firmly in
  the sky layer.
