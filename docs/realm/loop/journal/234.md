## the build

Three SVG edits, one verify-anim extension.

### 1. `windmill.svg` — dust fleck drift

The 4 existing dust circles (224-228 viewBox)
each gain `<animate>` opacity (0.32→0.55) +
`<animateTransform translate>` small drift.
Staggered begin times (0/-1s/-2s/-1.5s) and
varied dur (3.0/3.4/3.6/3.8s) so the flecks
don't pulse together. Each fleck drifts in a
slightly different direction (left/right/
up/down) for organic randomness.

### 2. `market.svg` — striped awning sway

Wraps the awning trapezoid + 4 white stripes
+ scalloped fringe in a parent `<g>` with
`<animateTransform type="rotate">` ±2° around
the top-edge center (64, 84). 4.5s asymmetric
cycle (-2° to +1.5°) reads as wind from one
direction. Top-edge highlight stays OUTSIDE
the sway group (anchored to the actual top
line so the sunlit highlight doesn't move).

### 3. `blacksmith.svg` — forge fire pulse

Three `<animate>` elements on the forge
window:
- Outer glow ellipse: opacity 0.85→1.0
  with random-feeling key-times.
- Fire core ellipse: rx 2→2.3 (subtle
  scaling — flame breathes).
- Inner highlight: opacity 1→0.95 (tracks
  glow phase).

All three share `dur="1.6s"` so the pulse
reads as ONE breathing flame, not three
elements oscillating independently. Random
key-times prevent metronomic feel.

### 4. `verify-anim.mjs` extension

Added market + blacksmith to SPRITES list.
Now 7 sprites verified (windmill / castle /
tower / house / bakery / market / blacksmith).
