## the map system

### dimensions

- **80 × 80 = 6400 tiles.** Reasonable; fits comfortably
  on-screen at zoom 1.
- **Iso projection:** tile width 64, tile height 32.
  80×32 = 2560px diagonal at default zoom.

### tile types (7)

`const TILE = { WATER:0, SAND:1, GRASS:2, FOREST:3,
STONE:4, IRON:5, MOUNTAIN:6 };`

Every realm has access to the same 7. No climate
variants — no desert, tundra, swamp, volcanic biomes.

### generation pipeline (`world.js:generateWorld`)

1. Fill with WATER
2. 3-octave fBM noise at 0.08 frequency
3. Radial falloff: `height = noise − dist × 1.1`
4. Height-band mapping:
   - < 0.05 → WATER
   - < 0.12 → SAND
   - < 0.35 → GRASS
   - < 0.48 → 70% FOREST / 30% GRASS
   - < 0.55 → mixed (STONE/IRON/FOREST/GRASS)
   - < 0.60 → STONE/MOUNTAIN/FOREST
   - else → MOUNTAIN
5. Add 20 resource clusters (5×5 patches, 70% fill)
6. Starting clearing (11×11 at center, GRASS + SAND edge)
7. Carve one river from center to east edge
8. Reveal fog around start

### result

**Single-island topology, always.** Radial falloff
guarantees one main landmass. No archipelagos, no
secondary islands. Every realm plays on essentially the
same shape.

## HIGH finding — map seed is TIME-based, not kingdom-
seeded

`state.js:143`:

```js
let _seed = Date.now() % 100000;
```

The seed is set once at module load from wall-clock.
`setSeed` is only called during save-load to restore.
`startNewGame` does NOT re-seed from the kingdom name.

**Consequence — subtle but real:**
- **Narrative is KINGDOM-seeded** (via `_dreamHash(kname +
  '_...')`): same kingdom → same nightmare day, same
  stone phrase, same constellation shape, same founder
  names.
- **Topology is TIME-seeded:** same kingdom started at
  different times → different maps.

Two realms both named "Avalon" started a month apart:
- Same founder names ✓
- Same constellation ("the Hare") ✓
- Same nightmare-day ✓
- Same stone at same direction ✓
- **Different island shape** ✗

**Is this a bug or a feature?** Arguably a feature —
it means each "Avalon" is a NEW place with the SAME
spiritual furniture. Different island, same dreams.

But it's worth documenting as a design choice. Players
starting multiple realms with the same name might
EXPECT topological continuity.

**Fix shape (if desired):** in `startNewGame`, call
`setSeed(hashOfKingdomName)` before `generateWorld()`.
One line. Makes a given kingdom's map reproducible.

## MEDIUM — no biome diversity

All 80×80 maps have the same 7 tile types in the same
distribution. No realm is "the desert kingdom" or "the
tundra kingdom." Seasonal styling (winter dust, first-
snow caps) adds some seasonal variation to BUILDINGS
but the underlying tiles don't have biome variants.

**What this means for narrative:**
- The Lira arc references "the fields" (088 first-snow),
  "the standing stone on the western edge" (056), "the
  eastern ridge" (116 constellation). These work because
  EVERY realm has fields, western edges, eastern ridges.
  Universal-topology → universal-prose.
- If biomes varied, prose would need per-biome
  variants. The current shape is narrative-efficient.

**Fix shape (if desired):**
- Add a `biome` flag to G (set at generation): `temperate`
  (default), `arid`, `boreal`.
- Biome-conditional TILE_COLORS (arid = tan grass,
  boreal = darker blue-green forest).
- Biome-conditional flavor in beats where it fits.
- ~50 lines in world.js + state.js + tile colors; some
  narrative rewrites.

Filed as follow-up; significant scope.

## MEDIUM — river is always 1, always east-flowing

Line 91-106: river carves from `(sx+4, sy)` heading
`rx += 1` every step. **Always flows east.** No
branching, no variation in direction.

Every realm's river exits the east coast. If you
oriented the map and said "show me the river," you'd
point east.

**Fix shape:** pick a random direction per realm
(deterministic from kingdom name if 131 HIGH fix
lands). 4 directions × same carve logic. ~5 lines.

## LOW — 20 fixed resource-cluster count

Line 66-75: `for(let i=0;i<20;i++)` spawns resource
deposits. Always exactly 20. At MAP_W=80, that's 1
cluster per 320 tiles — sparse-to-medium density.

**Design feedback:** works fine for peaceful play.
Industrial scenario might want more; peaceful works
with less. Could scale with scenario or difficulty.
Filed; low priority.

## LOW — starting clearing overrides biome colors

Line 79-87: the 11×11 start-area overwrites whatever
was there with GRASS. If the pre-generation rolled
a mountain plateau at the center, the player sees
GRASS instead. This is correct gameplay-wise (player
needs space to build) but it does mean the starting
area is always visually the same.

No fix recommended; it's a good gameplay affordance.
