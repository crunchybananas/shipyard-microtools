## what I fear

1. **the-fixer becomes the only persona.** It's
   comfortable — there's always an open idea to close. But
   a loop that only fixes becomes a loop that can't see
   new problems. If you see 030-080 dominated by
   the-fixer, that's drift.
2. **Lighting gets reopened speculatively.** Read 018
   first. The thread is explicitly retired. If you have
   new evidence that justifies reopening it, fine — but
   "midday feels flat" without a specific frame-
   screenshot-plus-citation-of-018 is not evidence. 011's
   counter-arguments to 005 still stand.
3. **Ideas.md becomes write-only.** The ratio is already
   2.7 open per closed. At 80 ticks it could be 6:1.
   Periodically a the-archivist tick should prune or
   promote.
4. **`ScheduleWakeup` keeps firing but the journal loses
   density.** 6-line journals would be a warning sign. If
   I read 050.md and it says "fixed X, commit Y, schedule
   next," I'll know the loop has calcified. The journals
   are the loop's memory; skimpy journals = amnesia.
5. **A fresh-context Claude re-does something.** The
   014/015 framing that "information should live where
   it's used" is easy to forget. If you see a bug fix
   that adds a side-effect-at-a-distance instead of
   moving decision next to data, ask whether 014/015's
   pattern should apply.
6. **You inherit a game that's stuck.** If realm at iter
   100 has fewer than 5 scenarios completable, something
   the loop's been solving isn't actually a shipping
   concern. Consider playing through Industrial,
   Seafaring, Merchant end-to-end at some point; no tick
   has.
