## the build

`docs/realm/js/story.js:_DREAM_THREADS` — 3 threads converted
from array-form images to function-form, each with a
founder-conditional variant gated on `founders_named`:

### founding (always-available, gets founder1)

```js
if (G.storyFlags.founders_named && G.storyFlags.founder1) {
  base.push(`${G.storyFlags.founder1} keeping the first fire alive, without effort or speech`);
}
```

Compounds with existing image #4 "three settlers at a fire
that will not go out" — the new variant names one of those
settlers (founder1) explicitly.

### hearth (requires house, gets founder2)

```js
if (G.storyFlags.founders_named && G.storyFlags.founder2) {
  base.push(`${G.storyFlags.founder2} standing at a door that opens on a room they do not recognize`);
}
```

Complements the existing "door that opens on a room the owner
has never entered" image — the founder IS the owner, and the
dream shows them not recognizing their own legacy.

### harvest (requires farm, gets founder3)

```js
if (G.storyFlags.founders_named && G.storyFlags.founder3) {
  base.push(`${G.storyFlags.founder3} counting grain by the handful, unable to remember how many handfuls`);
}
```

Echoes 093's winter arc ("the way elders count") — founder3
counting by hand is the same temporal sensibility as founder1
counting winters. The founders share a gesture of memory across
seasons.

## why this shape

- **One founder per thread, not all founders in each.** Keeps
  each thread's tone distinct — founder1 owns fire, founder2
  owns home/door, founder3 owns harvest/counting. Three
  parallel narratives; a 100-day run will see each surface
  occasionally.
- **Appended to existing pool, not replacing.** The new image
  joins the existing 4-5 images. Weight per pick: 1/6 for
  founding, 1/5 for hearth, 1/6 for harvest. Founder image
  won't dominate; just thickens the pool.
- **Function-form images pattern already established** —
  forge/market/learning already use `images: (G) => [...]`.
  097 extends the pattern to founding/hearth/harvest without
  introducing a new convention.
- **Gracefully degrades** — realms where 072's founder-naming
  hasn't fired (rare: founders fire day 3-6, dreams start
  day 10) see the original 4-5 images only. No crash, no
  empty-variable prose.
- **Tone-matched to 089 nightmare** — those founder-named
  images used an uncanny register ("footprints leading only
  away"). 097's dream variants use a dream register —
  effortless presence ("without effort or speech"), uncanny
  recognition ("do not recognize"), memorial counting
  ("unable to remember how many").
- **Continues the 088/089/093 arc** — founder1 is now the
  canonical MEMORY-bearer: first-snow, first-nightmare-
  reference, winter-count-elder, first-fire-keeper. Five
  surfaces where founder1 is the central figure.
