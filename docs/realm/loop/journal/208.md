## why this shape

- **Register-vs-tag distinction is the load-
  bearing insight.** Future authors picking a
  new beat will think first about REGISTER
  (what voice does this want?) then TAG (what
  echo/eviction behavior?). Conflating them led
  to 207's "11 misc-tag entries" mis-claim.
  Making the orthogonality explicit prevents
  future ticks from making the same mistake.
- **Sub-types are descriptive, not exclusive.**
  207 is early-game-mood AND land-as-agent.
  193 is naming-place AND observational-elder.
  Forcing exclusive categorization would require
  a primary-vs-secondary axis that the cluster
  doesn't actually need. Per 075 doc discipline:
  describe what's there, not what would be tidy.
- **Land-as-agent filed within the cluster
  doc, not separately.** 207's filing said
  "candidate observed-pattern (1 use)." 208
  could have created a 6th `## observed
  patterns` entry; instead it nests land-as-
  agent UNDER the cluster doc as a sub-type.
  Reasoning: land-as-agent is a sub-shape OF
  the observational-elder register, not a
  separate observation. Same promotion rule
  (3+ uses); easier to find when filed near
  its semantic neighbors.
- **Same-tick discipline for ALL doc locations.**
  204 [process] said "update invariants in the
  same tick when shipping invariant-governed
  code." 208 extends this to OBSERVED-PATTERNS
  too — when shipping a beat that touches
  multiple doc entries, update all of them.
  206 rival caught the invariants section but
  missed the observed-patterns mirror; 208
  catches the mirror.
- **Cluster doc is descriptive, not
  prescriptive.** Per 203 demotion of 188:
  observed patterns describe; invariants
  prescribe. The cluster doc gives authoring
  GUIDANCE (still hopeful-or-resigned register;
  static prose default) but doesn't force any
  shape. The 203 positive rule ("surprise the
  realm, not satisfy a pattern") is referenced.
