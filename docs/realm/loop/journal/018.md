## the 6 frames (012+017 pipeline)

1. **Summer midday** (`t=0.50`, summer). Overlay off. Pure
   summer tileShift `[8,5,-5]`. Saturated warm green,
   golden sand SE, inky teal water. Bright and vibrant.
2. **Summer dawn** (`t=0.05`, summer). Soft rose overlay
   on top of warm summer tiles. Slightly muted; the rose
   and the tile-warmth overlap into a gentle morning
   glow.
3. **Summer dusk** (`t=0.70`, summer). Amber overlay on
   warm summer tiles. Olive-warm grass, deeply golden
   sand, water reads warm-teal. **Still the strongest
   frame.**
4. **Summer night** (`t=0.95`, summer). Cool indigo
   overlay. Dark navy-green grass, inky blue water,
   stars in sky. Classic night.
5a. **Winter midday** (`t=0.50`, winter). Overlay off.
    Tuned winter tileShift `[-5,-3,+8]`. Grass reads
    *dormant cool green* — not the old drained wash.
    Water blue but not milky. Sand warm-tan.
5b. **Winter dusk** (`t=0.70`, winter). Amber overlay on
    the newly-cooler winter tiles. Grass darker-olive,
    sand golden, water subdued blue. Distinct from 5a
    while sharing the same base.

## new weakest frame

With 5a rehabilitated, the weakest slot moves to:

**Summer dawn (Frame 2)** — the rose tint is real but
rides on a warm summer palette, so it reads less as
"distinct dawn" and more as "slightly tinted summer
morning." The dawn window itself (t=0-0.10) is short and
starts with the darkness curve already ramping up from
its floor, so overlay magnitude never gets high. There's
no dawn equivalent of dusk's t=0.70 peak moment.

Is this a bug or a feature? Both, honestly.

- **Feature**: real dawns ARE short and gentle; a game
  that makes dawn as dramatic as dusk would read as
  theatrical, not a morning.
- **Bug-adjacent**: the dawn profile (rose) isn't
  particularly readable at any moment. Whereas dusk has
  a clean amber identity, dawn feels like "summer plus a
  hint of warm."

If a future tick wanted to push on this, the options are:

1. Widen the dawn window (e.g., `t < 0.15` instead of
   `t < 0.10`) so the rose has more room.
2. Strengthen the rose profile (shift `kR` lower, `kG`
   similar, `kB` unchanged).
3. Accept dawn is quiet by design and call this frame
   "good enough."

No action this tick. Filing as an observation.

## what I noticed (graphics, broader)

1. **The gallery is coherent.** Six frames across time-
   of-day + season, and each has a distinct identity.
   Before 012+017 the gallery had two frames (5a, 5b)
   that bled into each other. Now each frame has its
   own tonal fingerprint. That's the structural win.
2. **The strongest frame is still Summer dusk.** If
   realm wants a cover-art screenshot, t=0.70 + summer
   is the recipe. Worth codifying this into photo-mode
   preset when photo-mode ships.
3. **Winter gained a personality.** Pre-017, winter
   midday was the "frame everyone avoided." Now 5a is
   legitimately pleasant — a muted, reserved, quiet
   frame that reads winter without reading "broken."
   That changes how winter *feels* to play.
4. **HUD bug persists**: `G.season` live-override
   still doesn't propagate to the HUD (reads "Spring"
   throughout despite `G.season='winter'`). 003 finding,
   not resolved. Won't affect real play since real play
   advances via `updateSeason()` which writes to the
   same key via a different path. But a future
   `G.setSeason()` helper would close this.
