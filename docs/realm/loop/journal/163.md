
Faceted cylinder (default 8-sided reads as round at
isometric scale). Per-facet outward normal computed
at the midpoint angle so the shader's `dot(N,
uLightDir)` produces correct lighting per face.
Top cap as triangle fan from center.

22-line addition. First new `Mesh` primitive since
the file was created.

## why this shape

- **Same building as 158 (canvas) and 161 (SVG)**
  — `granary` is now the first building expressed
  across all three render layers. Future ticks
  can compare the same form rendered three ways.
- **First new `Mesh` primitive** since file
  creation. The existing primitives (pushBox,
  pushPyramid, pushQuad, pushTri) are sufficient
  for cubic+pyramidal forms but can't express
  cylinders cleanly. Adding `pushCylinder`
  unblocks future round buildings (well, tower
  could benefit, fountain if added, etc).
- **8-sided default** is a deliberate iso-scale
  choice. 4 sides is square; 6 is hexagonal; 8
  is the threshold where flat-shaded faceting
  reads as round at typical iso zooms. Could
  raise to 12 or 16 if a single hero building
  needs smoother curve, but vertex count grows
  linearly.
- **Dome via pyramid stack** matches existing
  tree canopy idiom — proven to read as
  organic-curve at this resolution. A proper
  spherical mesh (lat/long subdivision) would
  cost ~80+ vertices for similar visual quality.
- **Cleanup paired with feature add.** Removing
  the debug pillars on the same tick keeps the
  diff coherent: this was the first deliberate
  3D tick, so doing the cleanup now signals
  "the loop is taking ownership of this file."
  Future 3D ticks start from a clean baseline.
