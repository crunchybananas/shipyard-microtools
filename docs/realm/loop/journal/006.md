## the actual fix

Pivoted to the next-oldest un-done item: idea 002 "welcome-banner
overlaps starting citizens at T+0–3."

Root cause: `#toast` in `index.html:153` was positioned
`top:50%; left:50%; transform:translate(-50%,-50%)` — dead
center of the viewport. Since `notify('Welcome to Realm...')`
fires at game-start (main.js:123) with a 2.8-second `show`
duration, the toast sits directly on top of where the starting
3 citizens spawn.

The toast CSS is shared by *all* notifications — info, danger,
event, mission. So "shortening just the welcome" would require
special-casing the callsite. A structural move is cleaner:
reposition the shared toast element to top-center.

Changes to `index.html`:

1. `#toast`: `top:50%` → `top:5rem`; `transform:translate(-50%,-50%)`
   → `transform:translateX(-50%)`. Now anchors ~80px from the
   top, below the HUD, horizontally centered.
2. `@keyframes toastShake`: dropped the Y component from each
   frame so the shake animation still works with the new
   horizontal-only transform.

No JS change. No new elements. Two CSS edits in one line each.
