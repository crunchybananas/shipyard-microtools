## what church.svg expresses that canvas struggles with

- **Gothic stained glass.** Canvas drawChurch uses
  hand-coded radial+linear arcs to approximate
  arched windows; the SVG uses proper `<path>`
  commands with elliptical arc segments
  (`A 3 3 0 0 1 ...`), giving genuine arch shapes
  instead of canvas-`arc()`-half-circle
  approximations.
- **Lead tracery.** The vertical mullion +
  horizontal transom dividing the rose window into
  4 panes is a single line per tracery element in
  SVG; canvas would need a series of `moveTo`/
  `lineTo` calls and re-strokes.
- **Stone-block pattern fill** for both nave AND
  steeple. Canvas drawChurch does horizontal mortar
  lines via for-loop; SVG declares the pattern
  once (`<pattern id="churchBlocks">`) and
  instances it twice via `fill="url(#churchBlocks)"`.
- **Bell with proper highlight.** The bell ellipse
  with off-center radial highlight reads as
  metal-with-light-source. Canvas would do this
  with a `createRadialGradient` allocated each
  frame.
- **Crisp at photo-mode zoom.** The cross finial
  (1×6 px in viewBox) and door ring handle
  (0.25 px circle) stay sharp at 5× SVG zoom
  where canvas pixel-stairs.

## what I changed in form vs canvas

- Canvas drawChurch has **kingdom-hashed glass
  palette** (4 variants: classic blue / rose /
  amber / monastic green) and **kingdom-hashed
  steeple ornament** (bell / weathervane / saint
  statue). SVG ships **classic blue + bell only**.
  Variants would need separate sprite files (or a
  later inline-template tick where the integration
  picks variant via SVG fragment).
- Canvas has interior-light **spill on lower
  wall** as a vertical gradient; SVG has the same
  via `linearGradient` over a thin rect band — same
  effect, cleaner expression.
- Canvas does **subtle 4-pixel-spaced horizontal
  stone lines** for nave wall; SVG uses **pattern
  fill with 4-unit horizontal courses** — same
  underlying texture, vector-clean.
