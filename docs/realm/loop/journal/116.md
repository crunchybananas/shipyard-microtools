```js
{ flag: 'constellation_named', tag: 'milestone',
  trigger: G => G.season === 'autumn' && G.day >= 15,
  text: G => {
    const kname = G.kingdomName || 'Realm';
    const shapes = [
      'the Hare', 'the Ash-Tree', 'the Broken Wheel', 'the Three Sisters',
      'the Lantern', 'the Old Wolf', 'the Shieldmaiden', 'the Seven Sisters',
      'the Midwife', 'the Plough', 'the Harp', 'the Empty Hand',
      'the Crown', 'the Fisherman', 'the Smith', 'the Open Door',
      'the Warning', 'the Traveler', 'the Black Bird', 'the Sheaf',
    ];
    const shape = shapes[_dreamHash(`${kname}_constellation`) % shapes.length];
    const f = G.storyFlags.founder2;
    return f
      ? `The first autumn stars stand clear above the eastern ridge. ${f}, looking up, names them ${shape}. The realm takes the name.`
      : `The first autumn stars stand clear above the eastern ridge. The elders name the pattern ${shape}. The realm takes the name.`;
  } },
```

## why this shape

- **First autumn past day 15 = earliest around day 15 (year-
  1 autumn).** Autumn is days 15-21 each year. Fires once
  per realm via `constellation_named` flag.
- **20-shape pool** gives 20 distinct constellations. At
  uniform distribution, a player running 5 realms sees 5
  different constellation names. Each kingdom's shape is
  stable across reloads (deterministic per kingdom-name
  hash).
- **Shape names draw from medieval-rural iconography.** The
  Plough, The Sheaf, The Smith, The Midwife — none use
  classical-Greek names (Orion, Cassiopeia). Makes the
  realm feel pre-science, pre-telescope. "The settlers
  look up and NAME" is the operative word — they're not
  discovering Orion; they're making their own sky.
- **Tag: milestone (reuse per 075).** New tag would be
  wrong — no new category; just another once-per-realm
  observation.
- **founder2 as focal.** The loop's founder-arc has been
  heavily founder1-weighted:
  - founder1: 088 first-snow, 089 nightmare pool, 093 ×3
    winter arc, 097 dream hearth thread, 103 requiem
  - founder2: 089 nightmare pool, 097 dream hearth, 115
    founders-named audio (all 3 notes)
  - founder3: 089 nightmare pool, 097 dream harvest thread
    (1 surface)
  The realm was risking founder1-oversaturation (100's
  letter flagged this). 116 gives founder2 their OWN
  canonical lane: the one who looks up at stars. Like
  Lira stands in fields + counts winters + keeps the
  first fire + is last-spoken; founder2 is the naming-
  eye, the sky-watcher.
- **Text "The realm takes the name" closes the beat.**
  The shape is inscribed in the kingdom's memory. If a
  future tick surfaces constellations again (echo, visual
  render, etc.), the stored kingdom+shape mapping is
  recoverable via `_dreamHash(\`${kname}_constellation\`)
  % 20`.
- **Deterministic → testable + echo-eligible.** Tag:
  milestone is in `_ECHO_SOURCE_TAGS` so a 059 echo could
  resurface ~30 days later: "Old folk remember the first
  time they called the stars ${shape}."

## what this is / isn't

**Is:**
- An un-filed surprise, satisfying Evolution mandate
- A founder2-focused beat, rebalancing the founder arc
- A deterministic-but-varied narrative surface (20 shapes)
- A lifetime-scale beat (fires on first autumn past day
  15, unless the realm dies before then)

**Isn't:**
- Visual — no render-layer change. A future tick could
  draw the constellation in the night sky on that day.
- Audible — no audio cue.
- Mechanically consequential — doesn't affect gameplay.
- Cross-system compound — triggers on season + day only;
  no dependencies on other flags.
