## what shipped

Single-line change in `enhancements.js:4491`:

```diff
- if (G.enemies.length === 0 && _lastRaidDay > 0 && G.stats.raidsSurvived > 0) {
+ if (G.enemies.length === 0 && _lastRaidDay > 0) {
```

Dropped the `raidsSurvived > 0` gate. Post-014 that
counter only increments when `hasDefense` is true at
spawn-time, so a defenseless raid that kills citizens
and withdraws leaves `raidsSurvived = 0` — and the
summary never fired.

The message-picker below already had the right branch:

```js
} else if (losses > 0 && kills === 0) {
  msg = `Raid on day ${_lastRaidDay}: ${losses} fell, no foe answered. The survivors bury their own.`;
}
```

Just the outer gate was blocking it.

and a chronicle entry (per 124's original logic). And
crucially: raidsSurvived stays at 0, so 014's
correctness is preserved — losing a defenseless raid
still doesn't count toward the "Survive 5 raids"
mission.

disabled features in `enhancements.js`. Each has a
`return; // disabled` at the top with 10-50 lines of

Removing the dead bodies wouldn't break ordering
(register still fires), but it would override a
written design intent. If a future `the-dead-code-

either (a) get user sign-off to override the intent,
or (b) find dead code in files without such a
guideline.
