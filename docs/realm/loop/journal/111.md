## the build

### 1. `docs/realm/js/audio.js` — new `requiem` case

Added after the `nightmare` case:

```js
case 'requiem': {
  makeOsc(ctx, dest, 'sine',     196, 0.08,  0.02, 5.0, t);
  makeOsc(ctx, dest, 'sine',     392, 0.035, 0.02, 4.5, t);
  makeOsc(ctx, dest, 'sine',     588, 0.015, 0.02, 3.5, t);
  break;
}
```

**Sound design — inverse of nightmare (106):**
- **196 Hz** (G3) fundamental — low but cleaner than
  106's 64 Hz. Not a bass chord; a bell toll.
- **392 Hz** (G4, 2×) + **588 Hz** (~D5, 3×) — harmonic
  overtones matching a real-ish bell partial profile.
  No dissonance this time.
- **Pure sines** (not triangles like nightmare). Cleaner
  sustain; less buzzy.
- **Short attack 0.02s** (bell struck, not sneaking in) +
  **long decay 5.0s** (rings away into silence).
- **Low gain 0.08** — distant bell, not ceremonial.

**106 vs 111 acoustic contrast:**

| | nightmare (106) | requiem (111) |
|---|---|---|
| Wave | triangle | sine |
| Root | 64 Hz (C2) | 196 Hz (G3) |
| Dissonance | 4 Hz beating | pure harmonic |
| Attack | 0.5s (sneaks in) | 0.02s (bell strike) |
| Decay | 2.2s | 5.0s |
| Character | tense, ominous | clean, mournful |

### 2. `docs/realm/js/story.js` — wire audio into NARRATIVE_BEATS dispatch

Added tag-specific fire in the `checkStoryBeats` loop:

```js
for (const beat of NARRATIVE_BEATS) {
  if (!hasFlag(beat.flag) && beat.trigger(G)) {
    setFlag(beat.flag);
    const text = ...;
    chronicle(text, beat.tag);
    if (beat.tag === 'requiem') {
      try { if (_PLAY_SOUND) _PLAY_SOUND('requiem'); } catch (_e) {}
    }
  }
}
```

Reuses the `_PLAY_SOUND` late-bound import from 106. No
new import, no new schema field on NARRATIVE_BEATS
entries.

## why this shape

- **Single toll, not a sequence.** The realm is over. A
  church-bell sequence would imply ceremony; one toll
  implies inevitability. Matches the prose "The last
  fire goes out."
- **Inverse of nightmare, not copy.** Nightmare = 2 tones
  with dissonant interval (unstable); requiem = 3 tones
  with consonant ratios (stable). The realm's worst
  moment is UNSTABLE; its final moment is FINAL.
- **Tag-specific inline branch, not onFire callback
  field.** If 112+ adds sound to other tags (stone, etc.),
  we can still extend with more inline branches — or
  refactor to a callback when 3+ tags need it. Don't
  generalize until there's pressure.
- **Late-bound `_PLAY_SOUND` reuse.** 106 set up the
  pattern; 111 inherits without adding import
  infrastructure.
- **try/catch on the audio call.** Matches 106's
  defensive pattern. Even if audio fails, the chronicle
  entry already fired (chronicle is the contract).

  design depth for one beat. Filed as idea.
