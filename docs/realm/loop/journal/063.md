# 063 — the-fixer (060 HIGH echo-tag fix)

**Challenge:** `the-fixer` — accepted. Closes 060's
HIGH after 3 ticks sitting. 1-line correctness win.

060's archivist audit found that 059's
`_ECHO_SOURCE_TAGS` missed two legitimate origin-
class tags:

- **`event`** — events.js beats (cloaked stranger,
  plague death, rival emissaries, tribute demand)
- **`season`** — main.js:310 season-transition
  beats ("The snows melt…" / "Long days and warm
  winds…" etc.)

Without the fix, these beats could never become
memory — a player whose realm survived a plague
would never hear "old folk remember the plague."

## the fix

`docs/realm/js/story.js:~643` — 2 strings added
to the Set:

```js
const _ECHO_SOURCE_TAGS = new Set([
  'milestone', 'victory', 'character', 'birth',
  'dream', 'nightmare', 'stone', 'raid',
  'event', 'season',  // added in 063
]);
```

Plus 6 lines of comment updating the rationale.
Net: +3 lines.

## what this means for ArchivistRealm-style runs

060's audit run showed `event` and `season` both
at 0 in the programmatic test — because main.js's
season-advance path isn't driven by
`checkStoryBeats` alone and events.js fires via
its own tick path. In real play:

- Season changes fire 4 times per game-year
- Events fire ~1-3 times per 100 days depending
  on luck

So a real 200-day realm might see:
- 4–8 season-transition beats
- 2–6 event beats

All 10–14 of those are now eligible echo sources.
With 059's 2% per-dawn rate and the 14-day age
gate, this roughly doubles the memorable-beat
pool for mid-late-game. Echoes will feel more
varied.
