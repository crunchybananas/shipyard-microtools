## HIGH — no defined type scale; near-duplicate sizes everywhere

A typical design system uses **6-8 tiers** with
deliberate spacing. The realm has **26 unique values**,
grouped into 4 obvious clusters of near-duplicates:

- **0.60 / 0.62 / 0.65 / 0.67 / 0.68** — 5 values within
  13% of each other. Users perceive these as the same
  size. Used interchangeably across HUD labels, build
  buttons, tooltips, missions.
- **0.70 / 0.72 / 0.75** — 3 values, within 7% of each
  other. Split between panels / chronicle / chips.
- **0.78 / 0.80 / 0.82 / 0.85** — 4 values, body tier.
- **0.88 / 0.90 / 0.92 / 0.95** — 4 values, upper body.

**Impact:** maintenance burden. A new tick wanting to
match "existing small-text" has no single authoritative
size to copy; the author eyeballs what looks right and
adds a 27th value.

**Fix shape:** define `:root` CSS variables:
```css
:root {
  --fs-xs: 0.65rem;   /* meta/labels */
  --fs-sm: 0.75rem;   /* small body */
  --fs-base: 0.82rem; /* HUD body */
  --fs-md: 0.95rem;   /* emphasis */
  --fs-lg: 1.1rem;    /* subtitles */
  --fs-xl: 1.4rem;    /* panel titles */
  --fs-2xl: 1.8rem;   /* section headings */
}
```

Then migrate existing usages. Large-scope refactor (~100
replacements across index.html) but mechanical. Filed
HIGH for a future the-re-shipper or refactor-only tick.

## MEDIUM — low-opacity body text may fail WCAG AA

Many labels use `color:rgba(255,255,255,0.4)` to
`rgba(255,255,255,0.5)` against the dark `var(--panel)`
background. Examples:
- `.res-lbl` color rgba(255,255,255,0.5) @ 0.62rem
- `.build-btn .cost` color rgba(255,255,255,0.5)
- `.title-section-label` color rgba(255,255,255,0.4)

WCAG AA requires contrast ≥ 4.5:1 for normal text. A
50% white on ~5% white panel-bg lands around 3.2:1 —
below AA. The text is READABLE but not compliant.

This pairs with 109 mobile audit HIGH (`user-scalable=no`)
and 107 accessibility-reviewer findings.

**Fix shape:** raise body-text opacities from 0.4/0.5
to 0.6/0.65. Headings and emphasized text already sit
at 0.7+ which passes AA.

## MEDIUM — chronicle prose readability is ACTUALLY GOOD

The beat prose (chronicle panel) is the most important
reading surface — 150+ strings audited in 118 critic
tick.

Current styling:
- `.chron-row` font-size: 0.8rem
- color: rgba(255,255,255,0.8)
- line-height: 1.45
- font-family: Georgia serif (different from UI sans)

Contrast: 0.8 alpha white on dark panel ~ 8.5:1, well
above AA. Georgia serif at 0.8rem with 1.45 line-height
is excellent for paragraph reading. **This is the best
typography in the game.** It's telling that the SMALLEST
surface got the most deliberate treatment — 118's beat
arc relies on it.

No change needed.

## LOW — title-screen type scale is generous but isolated

`.title-name` at 5rem, `.title-icon` at 5rem, `.title-
sub` at 1.1rem, `.title-desc` at 0.9rem — coherent
display scale for the title screen specifically.

Doesn't leak into in-game UI. Good separation.

## LOW — monospace used consistently

`.build-btn-shortcut` uses `'SF Mono',Monaco,monospace`
@ 0.55rem. `.mission-progress` uses `tabular-nums`.
These number-adjacent places use mono correctly. No
drift.

## LOW — 5-level heading hierarchy across panels

Panel headings (h3, h4, section labels) use:
- 0.65rem TEXT-TRANSFORM uppercase letterspacing-0.05em
  (section labels)
- 0.8rem bold (h3 in missions)
- 0.85rem bold (raid-banner, title-btn)
- 0.9rem regular (title-desc)
- 1.1rem (title-sub)

Consistent hierarchy: labels < h3 < h2 < display.
Works. Could benefit from the var-based scale above.

## honest conclusion

Typography is **not broken** — it's drifted. The realm
looks fine and reads fine. But after 130 ticks of
additive work, 26 near-duplicate font sizes is a
symptom of "author eyeballs it" rather than "author
pulls from a scale."

**None of the drift blocks shipping.** But a refactor-
only tick consolidating to 6-8 variables would make
every FUTURE typography-touching tick simpler AND let
a future designer (or critic tick 260) change the
whole visual rhythm with 6 variable edits instead of
150 CSS-line edits.
