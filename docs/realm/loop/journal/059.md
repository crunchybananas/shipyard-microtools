# 059 — chronicle-echo (surprise, new system)

**Challenge:** surprise-shaped, from 056's ideas.
Pitched there as "low-rate resurfacing of old beats."
The realm was origin-heavy — dozens of first-build
beats, cycling dreams, once-per-realm stone +
nightmare — but once any beat fired, it never came
back. Realm had events but no memory of them.

056's idea asked for a loop-back. 059 ships it:
`checkEchoBeat`, a new narrative surface where at
dawn, a small chance rolls a prior memorable beat
and wraps it in a memory-frame. Tagged 'echo' so
future UI / audit passes can treat it distinctly.

Evolution-mandate legitimate — a system nobody has
shipped yet, compounds the existing narrative
stack (039 dreams, 043 nightmare, 044 time-aware,
048 walker, 054 pool, 056 stone, 057 completed
build beats, 058 stone-render).

## the build

`docs/realm/js/story.js` — new `checkEchoBeat()`
wired into `checkStoryBeats` after the 056 stone
check.

**Selection rules:**
- Dawn only (`dayPhase < 120`) — matches the other
  beats' convention
- `G.day >= 30` — need some history for an echo to
  feel like distance, not repetition
- `G.chronicle.length >= 3` — need enough entries
  to have a meaningful candidate pool
- Candidates filtered to `_ECHO_SOURCE_TAGS`:
  milestone / victory / character / birth / dream
  / nightmare / stone / raid. Never echoes misc
  (elder-sayings) or prior echoes (would cascade)
- `c.day <= G.day - 14` — the source must be at
  least two weeks old, so an echo of a recent
  dream doesn't feel like the dream re-firing
- One echo max per day (`echo_${day}` flag)

**Probability:** ~2% per dawn via deterministic
hash `_dreamHash(${kname}_echo_${day}) % 100 < 2`.
Expected 3-4 echoes per 170-day realm.

**Framing:** 4 pre-written wrapper functions that
quote the source beat in full:

```js
const _ECHO_FRAMES = [
  (text) => `An elder is heard telling a child: "${text}"`,
  (text) => `Travelers at the market repeat an old tale — "${text}"`,
  (text) => `The bards still sing of that day: "${text}"`,
  (text) => `Old folk remember. "${text}" they say.`,
];
```

Quoting the full source preserves the original's
tense + voice; the frame just establishes "someone
in the realm recalls this." Simple and robust
without per-tag phrase rewriting.

## why this shape

Considered alternatives:

- **Higher rate (5-10%)** — too noisy. Echoes would
  become background hum.
- **Per-tag phrase rewriting** (e.g. "An elder
  remembered the day the forge was lit") — requires
  per-tag NLP-adjacent logic. Full-quote wrapping
  is simpler and respects the originals.
- **Targeted echoes** (always echo the nightmare,
  always echo the stone) — too predictable. Random
  selection lets the realm's "memory" feel
  organic.
- **Echo-of-echo cascades** — rejected. The 'echo'
  tag is excluded from sources specifically to
  prevent this.

The 14-day age gate is the key design choice. Without
it, an echo could fire on day 15 referencing day 14's
dream, which would read as the dream repeating rather
than being remembered.

## what this does to realm's narrative

The chronicle now has:
- **origin-class** tags (milestone, victory, birth)
  — fire once per event
- **cyclical** tags (dream, raid) — fire on
  schedule
- **once-per-realm** tags (nightmare, stone) —
  rare lifetime beats
- **echo** tag (NEW, 059) — a loopback surface
  that lets earlier beats re-enter the chronicle
  wearing someone else's voice

The effect over a 200-day realm: the earliest
founding beat might come back once as "the bards
still sing of the founding." The nightmare (day
150) has 50+ days of post-life potential where
echoes might reference it. The stone's inscription
("EchoTestRealm") can be re-quoted in a later
echo, giving the kingdom-name persistent presence.

Nobody shipped this before. It's the first
narrative-memory system the realm has.

## what I did NOT do

- **Rewrite source text per frame.** Tempting to
  paraphrase ("they say the first bell rang")
  instead of quoting directly. Would need per-tag
  logic and loses the original's voice. The quote-
  wrapping is the right call for v1.
- **Add an echo-specific chronicle filter in the UI.**
  Chronicle panel currently renders all tags;
  future UI work can visually distinguish 'echo'.
- **Connect echoes to the named-character system**
  beyond simple framing. An echo that surfaces a
  character's ensure-beat could use that character's
  name in the frame (e.g. "Rowen the Just was
  heard saying…"). Filed.
- **Persist echoes differently.** They use the
  same 300-entry cap as regular chronicle entries.
  If echoes run deep into long-session realms they
  could push older beats past the cap before they
  get echoed. Unlikely in practice (3-4 echoes per
  realm), but noted.
- **Add echo-coverage stats.** A future archivist
  tick could audit "how many original beats ever
  get echoed" — useful for the-archivist if the
  system gets tuned.
