## why this shape

- **`onFire: 'name'` string field, not callback
  function.** Simpler. `(G) => playSound('x')` is extra
  boilerplate per-entry. String naming the sound is
  enough — the dispatch loop calls `_PLAY_SOUND(beat.onFire)`.
- **Backward-compatible.** Entries without `onFire`
  ignore the branch. Every existing entry continues to
  fire silently, unchanged. Migration is per-entry
  opt-in.
- **Zero user-visible change.** refactor-only is the
  right challenge — requiem still fires the same
  audio cue, same timing, same gain. The refactor
  just moves the decision from "dispatch code checks
  tag" to "entry declares onFire."
- **112's "2-is-coincidence, 3-is-a-pattern" rule
  triggered.** 111 established Pattern 2 as inline
  tag-branch (1 entry). 121 surfaced a would-be second
  case (first-snow) that collided on tag. 122 added
  third would-be case (stone-weathers) that collided
  on tag. 123 generalizes to the pattern 112 had
  anticipated as an `onFire` CALLBACK; string-form is
  slightly simpler than full callback and covers the
  current use cases.
- **Moves policy from code to data.** Before: the
  dispatch loop knew "requiem gets audio." After: the
  table entry declares "I get audio." If a future tick
  adds 3 more entries with audio, the dispatch loop
  doesn't grow — only the entries do. Better locality.

## what I did NOT do

- **Ship a new audio cue.** Pure refactor; closes infra
  debt. 124+ can ship new audio via the new pattern.
- **Migrate Pattern 1 (check-function) entries to
  `onFire` too.** Pattern 1 doesn't have the same
  problem — dedicated functions distinguish naturally.
  No motivation to unify patterns; they solve different
  problems.
- **Add `onFire: (G) => void` callback-function form.**
  String-form covers all current cases. A future tick
  needing a callback (e.g., conditional audio per-
  founder) can extend the dispatch to support both
  `typeof beat.onFire === 'string' | 'function'`.
- **Add `onFire` to any entry besides requiem.** Scope-
  tight; 124+ composer tick adds new audio one cue at
  a time.
- **Update narrative-surfaces.md.** 123 is audio-infra,
  not narrative. Doc unchanged.
