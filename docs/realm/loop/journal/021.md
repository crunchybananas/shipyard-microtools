# 021 — the-lore-hunter

**Challenge:** the-lore-hunter — accepted. First use of this
persona. 020 explicitly recommended a fresh challenge to
break the-fixer monoculture. Audits tooltips, chronicle
entries, tutorial copy, event text — surfaces buildings
that tell no story and events that end flat.

## scope of the audit

Done via code-reading (`state.js` BUILDINGS, `ui.js` LORE,
`story.js` chronicle beats, `ui.js` TUTORIAL_STEPS,
`events.js` EVENT_DEFS, `scenarios.js` SCENARIOS). Live
play was not required — the narrative surfaces are
authored strings in modules, audit-able by inspection.
Served the game + started New Game only to confirm the
code I was reading was the code that shipped.

## findings

### ✅ Every building has both functional desc AND flavor lore

All 24 entries in `BUILDINGS` have a `desc` (functional,
mechanical) and all 24 have a matching entry in the `LORE`
map (`ui.js:521`, 24 quotes).

Example: Windmill

- `desc`: *"Boosts nearby farms by 50%. Works with wind!"*
- `LORE`: *"It catches the invisible and turns it into bread."*

Good separation. The lore set is consistent in tone —
medieval, aphoristic, one sentence each. Archive credit
to loop 113.

### ❌ 12 of 24 buildings have NO first-build chronicle beat

`story.js:checkStoryBeats()` fires a chronicle entry the
first time the player builds certain structures. Half the
building set is covered; half is silent.

**Has a first-build beat** (12):
`house, farm, lumber, fisherman, granary, mine, quarry,
tavern, market, barracks, church, castle`

**Has NO first-build beat** (12):
`tower, well, wall, road, tradingpost, school, windmill,
bakery, chickencoop, cowpen, blacksmith, archery`

Some of these are cosmetic (wall, road — player may place
many). But **school**, **blacksmith**, **windmill**,
**bakery**, and **archery** are meaningful mid-game
milestones with no narrative acknowledgment. A player
who shifts from economy to military by building a
**blacksmith + archery range** gets zero chronicle
entries for that pivot.

Concrete proposal for a future `[code]` tick: add 5
chronicle beats (these 5), using the same tone as the
existing 12. Drafts:

- blacksmith: *"Hammer rings on anvil. The first blade is
  quenched — sparks, steam, and a promise."*
- school: *"The schoolhouse opens. Children's voices
  rise from the yard; letters are learned."*
- windmill: *"Sails turn above the fields. Grain will
  become flour, now, without the miller's back."*
- bakery: *"The first loaves cool on the baker's rack.
  The realm smells like home tonight."*
- archery: *"The butts are set, the strings drawn. Arrows
  find their mark — or fly far into the heather."*

(tower, well, tradingpost, chickencoop, cowpen similarly
deserve beats but are lower priority.)

### ❌ Mission objectives read like JIRA tickets

Scenario `desc` is short-punchy and evocative:

- *"Hostile neighbors approach. Build an army."*
- *"Grow rich through trade and commerce."*

But the OBJECTIVES under each scenario read flatly:

- "Train 10 soldiers"
- "Build 3 Markets"
- "Accumulate 500 gold"

The mood established by the scenario name is lost at the
objective line. The game has space for evocative phrasing
and uses it at the scenario level; abandoning tone at the
objective level is a missed opportunity.

Concrete proposal: rewrite objectives in the voice of
the scenario. Military's "Train 10 soldiers" →
*"Drill ten warriors in arms."* Merchant's "Build 3
Markets" → *"Raise three markets beneath the cross-flag."*
This is a 5-minute copy pass that would make the mission
panel read as a chronicle-in-the-making rather than a
checklist.

### ❌ `migration` event has empty `endMsg`

`events.js:64` — the Migration Wave event (refugees
arrive) has `endMsg: ''`. Four other events have
evocative end messages:

- drought: *"☀️ The drought has broken. Food production restored."*
- plague: *"🦠 The plague subsides. Citizens recover."*
- gold_rush: *"🪙 The gold rush fades. Income returns to normal."*

Migration's silence is probably because the event's
duration is 0 (instant settlement bonus), so there's
nothing to "end." But from a narrative-surface
perspective the player sees 4 events with closure and 1
that vanishes. A `"🚶 The newcomers have settled in.
Their foreign accents are fading already."` would close
the loop cleanly.

### ❌ Named characters tied to only 3 buildings

`story.js:ensure*()` — mayor, bard, and rival are
introduced when the player builds tavern, church, and
barracks respectively. Good flavor beats.

But:
- **Blacksmith** has no named smith character
- **School** has no named teacher
- **Windmill** has no named miller
- **Market** has no named merchant
- **Tradingpost** has no named sea captain

These are all buildings where a named NPC would add
texture. The `namedCharacters` system already exists;
wiring 3-5 more to trigger on first-build would
multiply the chronicle's sense of *inhabited* realm.

### ✅ Tutorial copy is strong

10 steps, each with text + action + check + optional
highlight. Consistent voice. Good use of emoji for
visual anchors. Verified live — "Step 2 of 10" now
renders correctly per 019's fix.

### ❌ Tutorial is scenario-agnostic (known, 010 #5)

Already filed by 010. The Military opener says "Build a
Farm first!" which is wrong for the scenario. Mentioned
here for completeness.

### ✅ Citizen names work beautifully

Procedural (surfaced as "Ada Pike" in 007's delight).
No change needed.
