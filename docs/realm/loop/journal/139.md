## the build

Two small changes.

### 1. `docs/realm/js/save.js` — record + restore savedAt

In `saveGame`:
```js
state.savedAt = Date.now();
```

In `loadGame`:
```js
G._savedAt = s.savedAt;  // exposed on G for main.js to read
```

Legacy saves without `s.savedAt` leave `G._savedAt`
undefined — handled gracefully downstream.

### 2. `docs/realm/js/main.js:loadAndStart` — wait-time prefix

Extended 135's welcome-back to include a preamble when
real-world time has passed:

```js
let prefix = '';
const savedAt = G._savedAt;
if (savedAt) {
  const waitMs = Date.now() - savedAt;
  const waitDays = Math.floor(waitMs / (24 * 60 * 60 * 1000));
  if (waitDays >= 1) {
    prefix = `Your realm has been waiting ${waitDays} day${waitDays === 1 ? '' : 's'}. `;
  } else if (waitMs >= 60 * 60 * 1000) {
    prefix = 'Your realm has been waiting a few hours. ';
  }
}
notify(`${prefix}Where we left off (day ${last.day}): ${last.text}`, ...);
delete G._savedAt;  // one-shot
```

Three tiers:
- **< 1 hour** → no prefix (resume feels immediate)
- **1 hour to 24 hours** → "a few hours" (vague,
  non-specific — exact hour count feels clinical)
- **≥ 24 hours** → "N day(s)" (specific count, with
  correct pluralization — matches 092 year-grammar
  invariant)

`delete G._savedAt` after use: ensures the prefix
fires ONCE per resume, not on every subsequent
`visibilitychange` event.

## why this shape

- **Three-tier language.** Exact days/hours/minutes
  feels clinical and out-of-voice for the realm's
  pastoral register. "A few hours" + "N days" is
  warm. Real wait ≥ 1 month could get "a long time"
  but no tier for that yet (filed).
- **Singular/plural grammar.** `waitDays === 1 ? ''
  : 's'` matches the 092 grammar-fix pattern for
  year-milestones. Consistent.
- **One-shot `delete G._savedAt`.** If the player
  tabs away and returns (triggering 127's in-session
  welcome-back), we don't want 135's "Your realm has
  been waiting N days" to re-fire — they already got
  the cross-session version on initial load.
- **Graceful degradation.** Legacy saves without
  `savedAt` field show just the chronicle echo with
  no prefix. Zero regression.
- **Minimal coupling.** `G._savedAt` is a temporary
  session-only field; not meant to persist in G. The
  underscore prefix signals "private/internal."
  Deleted after use to avoid stale-state concerns.
