## the duplicates ARE dead code — which side wins?

Both paths check `if (!hasFlag(flag))` and both
call `setFlag`. Execution order in main.js loop:

```
updateCitizens()            ← various
updateProduction()          ← 
checkStoryBeats()           ← runs BUILDING_FIRST_BEATS first
…
enhUpdateAll()              ← runs enhancements' check* later
```

So **BUILDING_FIRST_BEATS wins** — the table-driven
beat fires, sets the flag, and enhancements.js's
duplicate silently no-ops. The enhancements.js
prose is unreachable code.

The pre-022-refactor prose in enhancements.js is
DEAD. Removing it would be a pure `cut-as-you-add`
follow-up.

some event beats (plague death, stranger, tribute),
but the actual DISASTER-event beats (plague sweeps
through, fire!, earthquake!, drought) live in
enhancements.js. Partial overlap; split
responsibility.
