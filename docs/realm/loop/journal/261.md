## the build

`docs/realm/js/main.js` — added a
`_applyRealmEndFilter()` helper +
integrated it into `gameLoop` (both
visible-tab and hidden-tab branches):

```js
let _lastRealmEndedApplied = null;
function _applyRealmEndFilter() {
  if (G.realmEnded === _lastRealmEndedApplied) return;
  _lastRealmEndedApplied = G.realmEnded;
  const filterStr = G.realmEnded ? 'grayscale(0.85) brightness(0.85)' : '';
  canvas.style.filter = filterStr;
  const post = document.getElementById('postfx');
  if (post) post.style.filter = filterStr;
}
```

CSS filter on BOTH the game canvas + the
postfx WebGL overlay covers the post-
process path. Tracked variable avoids
every-frame DOM writes (only flips on
state transition: live realm_fell OR
save load with realmEnded=true).

  characters legible, terrain colorful
- **Post (realmEnded=true):** muted
  near-greyscale tones, dimmed
  brightness, washed-out terrain — the
  whole world reads as somber/drained

Filter string `grayscale(0.85) brightness(0.85)`:
- 85% desaturation toward greyscale
  (15% color preserved for readability)
- 85% brightness (slightly dimmed —
  feels post-mortem without becoming
  unreadable)

## why these specific values

- **0.85 grayscale, not 1.0.** Fully
  desaturated would look like a screen
  filter rather than a tonal shift; 15%
  color residue preserves enough hue to
  keep buildings/terrain identifiable
  if the user is mid-action when the
  realm falls.
- **0.85 brightness, not 0.5.** Heavy
  dimming would feel like a "screen
  off" effect (broken/error state)
  rather than "world ended" (somber/
  reflective). Slight dim conveys the
  emotional weight without obscuring.
- **Both canvases.** The postfx WebGL
  overlay covers the game canvas when
  enabled. Filtering only one would
  leave the other with full color.
  Filtering both is robust regardless
  of which path is active.
- **Tracked variable.** gameLoop runs
  ~60fps; every-frame DOM writes are
  wasteful even if browser CSSOM
  diffing is efficient. State-change-
  only is hygiene.

## why CSS filter, not WebGL shader

- **Robustness.** Works regardless of
  postfx enabled state. WebGL2 might
  not be available on all
  browsers/devices; CSS filter is
  universal.
- **Surgical.** Single helper function
  + 2 lines in gameLoop. No shader
  edits, no uniform plumbing through 2
  call sites, no 30+ LoC postfx.js
  patch.
- **Composable.** CSS filter applies
  AFTER the postfx pipeline finishes,
  so any night-time tinting or season
  grading still happens — the realm-end
  desat layers on top as a final tonal
  override.
- **Reversible.** Setting filter to
  empty string fully restores; no
  per-shader state to reset.
