## why this shape

- **Exact-match only.** 143's precedent: "each panel
  migration picks exact/near-exact mappings, leaves
  design-shift ones for deliberate ticks." 150
  tightens to EXACT-only (no near-exact) because 3
  clusters in one tick makes boundary-creep more
  dangerous. Design shifts deferred.
- **Panel clusters.** Each cluster is a coherent
  surface with ≥2 migrations. Scattered single-rule
  migrations across many panels would pollute git
  blame with no payoff.
- **No `#missions h3` (0.8rem).** 0.8 → 0.82 is +2.5%
  (within perceptual threshold) but not exact;
  reserved for a future deliberate tick to keep this
  commit truly zero-visual-shift.
- **No `.mission-progress` (0.72rem).** 0.72 → 0.75
  is +4.2% (borderline); same reason.
- **No build-bar, tooltip, research-panel, tech-
  cards, trade-panel, achievements, etc.** Still
  many panels left. Phase 3 can pick them off.
- **Progress measurable.** Post-143: ~2 of ~150
  declarations migrated. Post-150: ~10 of ~155
  declarations migrated (~6.5%). Scanning the rest,
  there are ~15 more exact-match candidates for
  future phases.
