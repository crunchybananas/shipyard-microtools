# 088 — surprise (first-snow beat)

Design shape: small cross-system compound that
can only fire after three prior systems have
landed in the realm's history:

- **072 founders named** (conditional; produces
  the personalized variant)
- **season cycle** (must have reached winter)
- **087 winter visuals** (what the player sees
  when this beat fires)

## the build

`docs/realm/js/story.js` — ~10-line block inside
`checkStoryBeats`, just after the first-raid check:

```js
// Loop 088 (surprise): first-snow beat. Fires once, the first
// time the realm sees winter past day 10. References a founder
// by name if 072 has named them; otherwise generic.
if (!hasFlag('first_snow_seen') && G.season === 'winter' && G.day >= 10) {
  setFlag('first_snow_seen');
  const founder = G.storyFlags.founder1;
  const text = founder
    ? `The first snow falls on the realm. ${founder} stands in the fields a long time before going indoors.`
    : `The first snow falls on the realm. The settlers stand in the fields a long time before going indoors.`;
  chronicle(text, 'milestone');
}
```

Tag `milestone` (reused per 075's anti-tag-
proliferation invariant; the beat is a realm
milestone).

## why this shape

- **Day ≥ 10 gate** — protects against scenarios
  that might start in winter or weird save states.
  Matches the 071 happiness-threshold gate (pop ≥
  5) pattern: "let the realm settle before firing
  lifetime beats."
- **Founder conditional** — gracefully degrades.
  A realm whose founders haven't been named yet
  (seeded 3-6 day range, might beat happens
  first) falls back to "The settlers stand..."
  A realm whose founders ARE named gets the
  personalized variant.
- **Single beat, no variants** — matches 072
  founders (one entry, 3 names inline). The
  moment is precious enough to not need variant
  prose.
- **`milestone` tag** — in `_ECHO_SOURCE_TAGS`
  from day 1, so the beat is auto-echo-eligible.
  Future 059 echoes can resurface first-snow
  alongside other milestones.

## what this compounds

- **072 founders** — first beat to use a founder
  name by SPECIFIC REFERENCE (founder1). 079
  offering and 043 nightmare don't name a
  specific founder. 088 picks founder1 as
  "the one who stands in the fields" — arbitrary
  but consistent per realm.
- **087 winter visuals** — on the same day the
  first-snow beat fires, the player can scroll
  to farms and see them dusted with snow (087)
  and buildings with caps (loop 19+). The beat
  gives language to the visual.
- **season transition** (main.js:310 writes the
  season beat "Frost grips the land. Fires burn
  low in every hearth."). 088 fires
  IMMEDIATELY AFTER the season beat in the
  chronicle (next checkStoryBeats call). The
  combined narrative reads: "Frost grips the
  land..." / "The first snow falls on the
  realm. Lira stands in the fields..."
- **059 echo / 085 tag-filter** — 088's beat is
  milestone-tagged, echo-eligible, filterable.

## what I did NOT do

- **Fire at the exact moment of season
  transition** (coupling to main.js). Keeping
  the beat in story.js's `checkStoryBeats`
  means it fires on the NEXT dawn after winter
  begins, which is acceptable delay for a
  realm-scale memory. Would also be annoying
  fragile coupling across modules.
- **Reference all 3 founders.** Single-founder
  reference reads better narratively. "Lira,
  Torrin, and Maia all stand..." would be
  comic.
- **Variant texts.** 6-item pool like 079
  offering would be nice but overkill for a
  one-time beat. The prose works either way.
- **Year-specific snow** ("the third winter..."
  etc.). Each winter could potentially fire
  different beats. Filed as a future-extension
  idea.
- **Couple to 087's new farm/quarry dusting.**
  Beat fires regardless of which buildings the
  realm has. Feels universal.
