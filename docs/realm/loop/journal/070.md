## the fix

Simpler than 069's filed "add skip-list to
notifications.js" — main.js already has access to
`notify`'s `{ chronicle: false }` opt-out, added
at notifications.js:62-64 for exactly this
"don't pollute story history" case. Just use it.

`docs/realm/js/main.js:302` changed from:

```js
notify(`${s.name} begins! (Season ${seasonNum})`, 'event');
```

to:

```js
notify(`${s.name} begins! (Season ${seasonNum})`, 'event', { chronicle: false });
```

+ 6-line comment block documenting why.

Toast + activity feed still update (notify's
primary purposes). Only the secondary chronicle
write is suppressed. The direct
`chronicle(seasonTexts[G.season], 'season')` at
line 310 remains the one canonical chronicle
write per season change.

## why the simpler shape

069 filed "add 'season' to skip-list in
notifications.js:68's tagMap". Considered here.
Rejected because:

- The notifications.js code already has the
  `meta.chronicle = false` opt-out mechanism
  (loops 15's fix for an unrelated toast noise
  issue). The pattern exists; 070 just uses it.
- A caller-side opt-out is more explicit about
  intent at the call site. Reading main.js:302
  now tells you "this toast is intentionally not
  chronicled because line 310 below IS."
- A skip-list in notifications.js would be a
  parallel mechanism doing the same thing —
  would need maintenance as more tags want to
  skip.
