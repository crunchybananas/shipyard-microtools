### before

```js
let rx = sx + 4, ry = sy;
// ...
rx += 1;
ry += Math.round(rng() * 2 - 1);
```

- River starts 4 tiles east of center.
- Flows east (x++) with ±1 meander in y.
- Every realm's river exits the east coast.

### after

```js
const dir = Math.floor(rng() * 4);  // 0=E, 1=S, 2=W, 3=N
const [dxDir, dyDir] = [[1,0], [0,1], [-1,0], [0,-1]][dir];
let rx = sx + 4 * dxDir;
let ry = sy + 4 * dyDir;
// ...
rx += dxDir;
ry += dyDir;
if (dxDir !== 0) ry += Math.round(rng() * 2 - 1);  // horizontal → vertical meander
else             rx += Math.round(rng() * 2 - 1);  // vertical → horizontal meander
```

- Direction picked via `rng()` (132's kingdom-seeded).
- River start offset from center in chosen direction.
- Flow and meander both rotate by 90° per direction.

## why this shape

- **Direction hashed from 132's kingdom seed.** Because
  `rng()` reads `_seed` and `_seed` was set from
  kingdom name at `startNewGame`, the first 4 `rng()`
  calls inside `generateWorld` produce the same
  sequence for the same kingdom. The direction draw
  is deterministic.
- **Perpendicular meander** — the original code
  meandered y when flow was east. For north/south
  flow, meandering x is the right analog. The
  conditional `dxDir !== 0` swaps the meander axis
  automatically.
- **Start offset scales with direction** — `sx + 4 *
  dxDir` means north-flowing rivers START 4 tiles
  north of center (into the mountain-heavy high
  region). Same mathematical relationship as
  original, rotated.
- **No other generator changes** — height bands,
  resource clusters, fog reveal all unchanged.
- **Still single river per realm** — doesn't add more
  rivers. Just changes which direction the one river
  goes.

## what this changes perceptually

- Before: every saved realm's river is on the east side.
  Screenshots from 098 (photographer) would have all
  shown river-east. Players developing navigation
  intuition could say "the river is the eastern
  edge."
- After: each kingdom gets its own river orientation.
  Avalon might have a south-flowing river; Kestrel
  west-flowing. Adds topological fingerprinting.
  Players starting multiple realms see real variation.
- **Cross-realm memory:** someone returning to a realm
  named "Avalon" gets the same river orientation
  every time (thanks to 132's seed). Consistent.

## interaction with prior code

- **Starting clearing at center** (unchanged) — still
  11×11 grass regardless of river direction. If the
  river enters the starting clearing, it's carved
  through; if not, the clearing stays dry.
- **Fisherman's hut** requires TILE.SAND — river
  banks are SAND per the carve loop. A south- or
  north-flowing river creates sand tiles to the south/
  north of center, which affects fisherman placement.
  Players will discover new optimal fisherman spots
  per-kingdom.
- **Scenario difficulty** — no direct interaction.
  No scenario hardcodes east-coast fishing.
