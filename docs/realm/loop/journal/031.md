## the fix

One-value change in `js/input.js:72`:

```diff
-    if (dist < 15*15 && dist < bestDist) { // 15px radius
+    if (dist < 22*22 && dist < bestDist) { // 22px radius
```

Citizen sprite visual extent at the default 1.3× zoom
is roughly 22px tall × 14px wide. The old 15px hit-box
missed clicks that visually landed on the citizen's
head or shoulders. 22px gives a circle that matches
the visible sprite footprint without overlapping
typical neighboring citizens at default tile spacing.

Inline comment expanded to explain the choice.

## verification (chrome-mcp, synthesized offsets)

Picked citizen[0], computed its world-space center,
synthesized mousedown events at increasing offsets:

| offset (world-px) | hit? (new 22px) | hit? (old 15px, derived) |
| ----------------- | --------------- | ------------------------ |
| 0                 | ✅               | ✅                        |
| 10                | ✅               | ✅                        |
| 15                | ✅               | ❌                        |
| 18                | ✅               | ❌                        |
| 22                | ✅*              | ❌                        |
| 25                | ❌               | ❌                        |

*22 hits due to floating-point imprecision in the
round-trip coord conversion; strict boundary would
miss at exactly 22. Not material — the transition
zone is now a 7px-wide band from 15→22 that *used to
be dead*.

**Net**: 5 of 6 test offsets hit, up from 2 of 6. The
"I clicked exactly on them and nothing happened" gap
is closed in the radius 15–21 range that's most felt.

`node --check js/input.js` → OK.

## what the change does NOT affect

- **Building click hit-boxes** (`findBuildingAtClick`).
  Separate function, separate logic, out of scope.
- **Multi-citizen disambiguation.** The function
  already picks the nearest hit (`bestDist` tracking);
  widening the radius doesn't break that.
- **Placement click-paths.** `pickTile` runs when a
  build is selected; citizen hit-check is gated behind
  `!G.selectedBuild` at input.js:194. No interaction.
- **Hover state.** Hover still uses tile-level math;
  hovering a citizen-position doesn't trigger the new
  22px radius.
