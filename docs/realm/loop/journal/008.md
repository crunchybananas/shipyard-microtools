# 008 — the-fixer (scenario descriptions)

**Challenge:** the-fixer — accepted: pick 007's #1 novice finding
(scenario descriptions missing on landing) and close it. Smallest
available [code] tick with a clear, concrete novice win.

## what

Added a `#scenario-desc` element below the scenario button row on
the landing page. Populates from `SCENARIOS[id].desc` via the
existing `window.setScenario(id)` dispatcher.

**The key insight**: the descriptions already existed in
`js/scenarios.js` as a `desc` field on every scenario object.
They were only used in-game (mission panel). Surfacing them on
landing was three small edits, not a data authoring pass:

1. `js/main.js:24` — extend the existing scenarios import to
   include `SCENARIOS`.
2. `js/main.js:146 setScenario()` — after the active-class
   toggle, look up `SCENARIOS.find(s => s.id === id).desc` and
   write it into `#scenario-desc`. Null-guarded.
3. `index.html` — add `#scenario-desc` CSS rule (small italic
   muted, `min-height:1em` to prevent layout jump) and insert
   the element with the Peaceful default text.

No duplicated strings. One source of truth (scenarios.js).

## verification

- `node --check js/main.js` → OK.
- Hard-reloaded in chrome-mcp. Landing page shows "A quiet
  island. Build your first settlement." beneath the scenario
  buttons in small italic muted text.
- Clicked Military → desc updated to "Hostile neighbors
  approach. Build an army."
- Programmatically iterated all 5 buttons; all 5 descriptions
  render correctly:
  - peaceful_start: "A quiet island. Build your first settlement."
  - military_rise: "Hostile neighbors approach. Build an army."
  - merchant_kingdom: "Grow rich through trade and commerce."
  - seafaring: "Master the seas through trade and fishing."
  - industrial: "Build a production empire."
- No regression visible in the title-screen layout — the
  `min-height:1em` holds space so switching between scenarios
  doesn't jump the kingdom-name input downward.

## what I noticed (graphics)

The desc element uses italic muted-white (0.55 opacity). It sits
in the same color family as the `.title-section-label` "SCENARIO"
header, which gives the landing-page hierarchy a legible
top→bottom scan: bright title → label-level header → button row →
flavor line → form inputs. **Graphic state improved slightly**:
the landing now *teaches* rather than just presents. Same
improvement as 006, this time at landing rather than in-game.

## design notes for future ticks

- **One source of truth matters.** If a future iteration wants
  to rewrite these descriptions, they only need to change
  `scenarios.js`. The landing page picks up the new copy for
  free on the next page load.
- **The descs are punchy but not yet *tempting*.** Current
  tone is factual ("Grow rich through trade and commerce.").
  A future `the-name-giver` or `the-art-director` could argue
  for more evocative copy ("Coin before conquest."). I didn't
  touch the copy because (a) that's a design call, not a fix,
  and (b) the scope of this tick was to *surface* what exists,
  not to rewrite it.
