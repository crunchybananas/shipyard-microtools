# 043 — a-scene-that-happens-once (NIGHTMARE)

**Challenge:** `a-scene-that-happens-once` — surprise
slot picked from 039's follow-up pool. Back on the
Evolution mandate after the 040-042 UX detour.

The idea: once in a realm's lifetime, on a seeded
day, the ordinary collective dream is replaced by
something wrong. Darker imagery, `nightmare` tag,
a toast so the moment is not missed.

## the build

One new function, one static image pool, one
character-conditional expansion, one late-bound
notify import — all in `docs/realm/js/story.js`.

Seeded so that:
- The day it fires is deterministic per kingdom
  (hash of `${kingdomName}_nightmare_day` → day
  50–250)
- The 3 images chosen are deterministic per kingdom
  (per-slot hash, de-duped)
- The flag `nightmare_fired` is set so it cannot
  repeat — ever, in that realm

Eleven base images in `_NIGHTMARE_IMAGES_STATIC`.
Four more appear if the matching named character
exists (mayor / smith / bard / teacher), so the
realm's own people bleed into the dream when
they've been named.

```js
export function checkNightmareBeat() {
  if (hasFlag('nightmare_fired')) return;
  const kname = G.kingdomName || 'Realm';
  const targetDay = _dreamHash(`${kname}_nightmare_day`) % 200 + 50;
  if (G.day !== targetDay) return;
  if (G.dayPhase > 120) return;  // dawn-only
  setFlag('nightmare_fired');

  const pool = _nightmareImagesForState(G);
  const rawImages = [];
  for (let i = 0; i < 3; i++) {
    const idx = _dreamHash(`${kname}_nightmare_img_${i}`) % pool.length;
    rawImages.push(pool[idx]);
  }
  const images = [];
  for (const img of rawImages) if (!images.includes(img)) images.push(img);

  const prose = `The realm does not wake — it is woken. ${images.join('; and behind it, ')}. The day begins with the feel of something ended.`;
  chronicle(prose, 'nightmare');

  try {
    if (_NIGHTMARE_NOTIFY) _NIGHTMARE_NOTIFY(`🌑 The realm did not wake well.`, 'danger', { chronicle: false });
  } catch (_e) {}
}
```

Hooked into `checkStoryBeats` right alongside the
regular dream check. Late-bound `_NIGHTMARE_NOTIFY`
dodges load-order coupling with notifications.js.

## why a nightmare (and not just more dreams)

039 shipped collective dreams as ambient recurring
texture. That system's strength is that it fires
often and ties the realm into a single felt story.
Its weakness is that it trains players to glaze: once
you've seen a few dream entries in the chronicle,
they become wallpaper.

A **once-per-realm** beat inverts the economics:
rarity itself is the signal. A player who plays long
enough to see this entry will remember it. A player
who doesn't will never know the game had it. Both are
fine — the beat is content for the long-tail, not for
every session.

The `nightmare` tag (distinct from `dream`) is
future-proofing. If chronicle search or tag filters
come later, the rarest beat in the game has its own
label.

## what I did NOT do

- **Trigger on in-game events** (e.g., "first raid
  loss triggers nightmare"). Tempting but wrong: it
  would make the nightmare feel earned / deserved. The
  point is it arrives without cause.
- **Add nightmare art / screen shake / audio sting**.
  The text prose + 🌑 toast is the beat. Anything more
  pushes toward horror-game theatrics. Realm is a
  pastoral settlement builder; the nightmare should
  read as *strange*, not as *scary*.
- **Tie nightmare day to a fixed real-world date**.
  That's the `time-aware` challenge from 039 ideas,
  which deserves its own tick.
- **Give the player a way to dismiss the flag**. No
  "new game plus" button, no debug clear. The beat is
  once. Debug menu could add one later if needed.

## trade-offs noted

The nightmare can collide with the regular dream on
that exact day — both gate on `dayPhase <= 120`. Right
now the nightmare runs first (called before
`checkDreamBeat` in `checkStoryBeats`), and since the
regular dream also has its own flag cycle, the day's
mood is just: nightmare, no normal dream. Good.

If a future tick reorders the beat calls, revisit.
