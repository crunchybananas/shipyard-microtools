## the invariants section is the real product

The "20 systems" list is a snapshot that rots
quickly. The **invariants** are the enduring
contract:

Example: "`BUILDING_FIRST_BEATS` is the single
source of truth for building-first beats. No
duplicate `updateXxxChronicle` in enhancements.js."

That sentence, if respected going forward, prevents
the 074 situation (5 ticks of dead code) from
recurring. Future ticks adding new building types
know exactly where to add the beat: the table.

Other high-value invariants:
- The execution-order section (enhancements runs
  AFTER story.js beats in the tick loop) — once
  documented, it stops being a surprise.
- The `{chronicle: false}` convention for
  notify+chronicle at same callsite — 070's
  pattern, now doc'd as the fix for duplicate
  toasts.
- The `_ECHO_SOURCE_TAGS` list — changes over time
  (059 initial, 063 added event/season, 065 added
  research). Doc points to the current live list
  so future expansion is local.

## what this tick doesn't cover

- **No game code changed.** Pure documentation.
- **The doc is ONLY loop-docs** — not player-
  facing. Players don't read loop/docs/.
- **No verification beyond grep matches** — the
  doc is about describing state, not changing it.
  If it's wrong in places, a future
  the-archivist tick can correct.
