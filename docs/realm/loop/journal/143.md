# 143 — refactor-only (first type-scale migration)

**Challenge:** refactor-only — accepted. Proves 140's
type-scale token infrastructure in practice. Picks the
highest-value target (chronicle panel — 130 flagged
the best typography in the game) and migrates the two
exact-match font-sizes to tokens.

## the change

`docs/realm/index.html` — 2 lines modified:

```css
/* before */
.chron-day-h { font-size:0.75rem; ... }
.chron-row   { font-size:0.8rem;  ... }

/* after */
.chron-day-h { font-size:var(--fs-sm); ... }
.chron-row   { font-size:var(--fs-base); ... }
```

## token mapping

| Selector | Before | After | Δ |
| --- | --- | --- | --- |
| `.chron-day-h` | 0.75rem | var(--fs-sm) = 0.75rem | 0 |
| `.chron-row` | 0.80rem | var(--fs-base) = 0.82rem | +0.02rem (+2.5%) |

**`.chron-day-h`** maps EXACTLY — zero size change.
**`.chron-row`** maps to the 0.78-0.85 cluster's
canonical token. +0.02rem at typical 13px base =
0.26px. **Below typical perceptual threshold**
(~2-4% for font-size). The chronicle prose reads
identically.

## why just these two

Chronicle panel has 3 font-sizes in the audited area:
- `.chron-day-h` — 0.75rem (migrated)
- `.chron-row` — 0.8rem (migrated)
- `.chron-chip` — 0.7rem (NOT migrated)

`.chron-chip` at 0.7rem would map to `--fs-sm`
(0.75rem) per 140's documented scheme. That's a +7%
shift — noticeable. Migrating it would consolidate
another tier BUT requires acknowledging a deliberate
visual adjustment. Outside refactor-only's "zero
user-visible change" envelope.

Leaving 0.7rem inline for now. A future tick can
migrate `.chron-chip` as a deliberate design-choice
tick (cluster consolidation) rather than a refactor.

## what this proves

- **140's tokens work in CSS** — browser resolves
  `var(--fs-sm)` → 0.75rem as declared.
- **Migration pattern is simple** — two-token swap,
  literal text replacement.
- **Risk is low when the mapping is exact.** Only
  the 0.02rem shift on `.chron-row` qualifies as
  "approximate"; still negligible.
- **Template for future ticks:** take a panel,
  identify which values map to existing tokens
  (exactly or ≤ 3%), migrate those, leave the rest.
