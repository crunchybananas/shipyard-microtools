## the build

New in `js/enhancements.js`, placed above the 048
wanderer block:

**`_shootingStarTick(logicalW, logicalH)`**
- Gate: `G.storyFlags.constellation_named` must be
  true (from 116)
- Gate: `!G.storyFlags.shooting_star_fired`
  (once-per-realm)
- Gate: `getDaylight() < 0.35` (deep night only)
- 0.3% spawn chance per eligible tick
- On spawn: `{ x, y, vx, vy, life: 28 }` streaming
  from upper-left quadrant toward lower-right
- On death (life → 0): set
  `G.storyFlags.shooting_star_fired = true`

**`renderShootingStar(ctx)`**
- Uses `globalCompositeOperation = 'screen'` like
  comets/ghost (additive brightness)
- Linear gradient tail from current position back
  2.5 × velocity (so tail length scales with speed)
- Bright head circle 1.8px with alpha ramping down
  by life remaining
- Fades with `nightStrength` (invisible at dawn)

Registered via `registerUpdater(_shootingStarTick,
true)` + `registerScreenRenderer(renderShootingStar)`
— no render.js edit required. `needsScreen: true`
ensures logicalW/logicalH pass through to updater.

Net LoC **+77** (all additive; no existing code
touched).

## why this shape

- **Third moment in the 116/151/159 arc.** 116
  names the realm's constellation; 151 draws it
  in the sky; 159 moves a star through the sky
  the realm named. Each tick adds a layer to the
  same night-sky vignette. This is what
  Evolution mandate wants — cross-reference over
  silo.
- **Respects 016's "out of place" concern.**
  File header at `enhancements.js:12` flagged
  meteors/comets/space-effects as removed because
  they felt generic. 159 resurrects the IDEA
  (shooting star) but GATES it so strictly that
  it's not sky-junk — it's a single moment per
  realm, tied to a specific narrative precedent.
  Same gating strategy as 151 (constellation
  only renders after being named).
- **Once per realm, persistent flag.** Uses
  `G.storyFlags.shooting_star_fired` so save/
  load preserves the "already fired" state.
  Most realms will see it; some long-lived
  realms may miss it entirely (0.3% per eligible
  tick means many deep nights can pass without
  a spawn). That's fine — miss-able is the
  philosophy (112 audio-surfaces invariant
  extended to visual realm).
- **No chronicle, no toast, no UI.** Purely
  visual. The event is its own acknowledgment
  — if the player sees it, they see it; if not,
  the realm simply had a shooting star one
  night nobody was watching.
- **Zero audio.** 154 skeptic halt in effect; no
  onFire, no playSound call. Even if a future
  tick wants to add audio, it should pass the
  live-audition gate first.
- **Velocity shape.** `(12+rand×6, 3+rand×2)`
  creates diagonal motion upper-left to lower-
  right at varying speeds. Life 28 ticks ≈ 0.47s
  at 60fps → star crosses ~420 px horizontally,
  ~100 px vertically. Roughly a diagonal slash
  across the upper-third of screen — classic
  shooting-star path.
- **Tail length = 2.5 × velocity.** Moving stars
  have longer tails; slower stars shorter. Feels
  physical.

## what I did NOT do

- **Multiple shooting stars per night.** One per
  realm, persistent flag. Anything more frequent
  edges toward the "space junk" 016 rejected.
- **Tie to specific constellation shape.** The
  star doesn't intentionally cross the named
  constellation's exact coordinates. It moves
  through the upper sky, which happens to be
  where the constellation lives. Overloading the
  shape coincidence would be forcing a symbolic
  moment the player won't necessarily notice.
- **Audio cue.** 154 halt honored. Even a quiet
  shimmer would violate the halt.
- **Chronicle entry.** The shooting star is
  visually self-acknowledging; adding prose
  would over-explain it.
- **Re-enable disabled comets/meteors.** 016's
  design decision stands for generic sky-junk.
  159 is the gated exception.
- **Modify constellation rendering.** 151's
  constellation is unchanged; 159 is a sibling
  layer.
