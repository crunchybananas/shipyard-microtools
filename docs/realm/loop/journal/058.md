## the build

Two files, both small edits.

### `docs/realm/js/story.js` — persist stone coords

In `checkStoneBeat`, after the beat fires, compute a
seeded tile location based on the direction variant
and store it in `G.storyFlags.stone_x` /
`stone_y`:

```js
const offset = _dreamHash(`${kname}_stone_offset`) % 36 + 22;
let sx, sy;
if (dir === 'northern')      { sx = offset; sy = 22; }
else if (dir === 'southern') { sx = offset; sy = 58; }
else if (dir === 'eastern')  { sx = 58; sy = offset; }
else                         { sx = 22; sy = offset; }
setFlag('stone_x', sx);
setFlag('stone_y', sy);
```

Range `[22, 58]` keeps the stone on the map's
interior (80×80 map, grass-likely tiles concentrate
around the middle 40). Deliberately not
grass-checked — stone on sand or near water is fine
for the narrative, and importing TILE from state.js
would bloat the story module. A follow-up tick can
add a snap-to-grass pass if needed.

Using `setFlag` with a numeric value works because
`setFlag` writes `G.storyFlags[key] = val`, not
just booleans.

### `docs/realm/js/enhancements.js` — render block

After 048's `_t048_draw`, a new `_stoneDraw` block
registered via the same aggregator pattern:

```js
function _stoneDraw(ctx) {
  if (!G.storyFlags?.stone_found) return;
  const tx = G.storyFlags.stone_x;
  const ty = G.storyFlags.stone_y;
  if (tx == null || ty == null) return;
  if (G.camera && G.camera.zoom < 0.6) return;  // hide at tiny zoom

  const p = toScreen(tx, ty);
  // Ground shadow
  // Stone trapezoid (wider at base) with gradient + edge highlight
  // ...
}
registerWorldRenderer(_stoneDraw);
```

Sprite geometry: ~5px wide at base, ~4px wide at
top, 12px tall. Trapezoid path (not rect) so it
reads as a proper menhir leaning slightly. Dark-
grey gradient (#4e4844 at top → #2e2826 at base).
Ground shadow ellipse. Left-edge highlight stroke
at 15% white alpha for sun-from-upper-right.

Hidden at `camera.zoom < 0.6` so distant views /
minimap-zoom don't clutter.

## what I did NOT do

- **Import TILE/MAP_W into story.js.** Preserves
  the module's light import footprint. Stone tile
  selection is heuristic (interior range 22-58)
  rather than grass-verified. If a future tick
  discovers stones landing on sand, snap-to-grass
  is filed.
- **Add a click handler or tooltip.** The stone is
  intentionally inert. Attentive players discover
  it; the chronicle entry is the only narrative
  surface.
- **Snap the stone to the nearest available grass
  tile.** Slight risk that the seeded tile lands
  on beach/water for edge-generated maps. The
  [22, 58] range mostly avoids this; the worst
  case is a stone at the shore edge which is
  still narratively fine.
- **Render the stone's inscription visibly.** The
  kingdom name is in the chronicle, not carved
  into the sprite at 5-px wide. Text would require
  a zoom ≥ 2 guard and decent font rendering — too
  much scope for a single-tick compound.
- **Add snow-cap on the stone in winter.** The
  buildings get this (render.js:2934); stones
  could but aren't structural to realm. Filed as
  a polish follow-up.
- **Animate or pulse.** Deliberately static —
  it's a menhir, not a beacon.
