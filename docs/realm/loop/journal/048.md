## the rule I set for myself

Something new in the game that:

- is visible to a player who pays attention at the
  right zoom
- has **no name** anywhere in the codebase: no
  string literal, no chronicle tag, no
  `G.namedCharacters` entry, no comment giving it
  away
- lives in module-local state (not `G`) so it
  doesn't show up in save files or debug handles
- doesn't fire any notification, toast, chronicle
  beat, or sound cue
- happens rarely enough that a player might miss it
  for many sessions

The concept name is intentionally off-limits inside
the repo. Whoever picks up the next surprise tick
OR plays the game at the right moment can decide
what it is.

## where it lives

`docs/realm/js/enhancements.js` — a new block at the
end, registered via the existing `registerUpdater`/
`registerWorldRenderer` pattern that loop 23+ uses.

Module-private identifiers only:

```js
let _t048 = null;
let _t048_season = null;
function _t048_tick() { ... }
function _t048_draw(ctx) { ... }
registerUpdater(_t048_tick);
registerWorldRenderer(_t048_draw);
```

Name scheme `_t048` was chosen because:

- the underscore prefix signals module-internal
- `t` is ambiguous (thing? token? trace?)
- `048` ties it to this tick without implying intent

A future iteration naming this will have to pick
*their* name. I'm not leaving a hint via variable
convention.

## behavior (for the journal — not the code)

Once per season (skipping winter), with ~55%
probability, a small dark silhouette begins at one
map edge and walks in a straight line to the
opposite edge. The path is deterministic from
kingdom name + season + day-bucket (so two
sessions in the same realm in the same season see
the same traversal if triggered). It only moves
while daylight > 0.35 — fades into night, resumes
at dawn. Expires on reaching its opposite edge.

Not rendered below `camera.zoom < 0.55` so it never
appears as a dot in the minimap-style view.
Rendered only via the world-transform renderer, so
it's drawn in world space between grass and
citizens (behind other moving figures).

Silhouette: 2 small dark rectangles stacked — a
body at `s.y-9..s.y-3` (2.4 px wide) and a head at
`s.y-12..s.y-8.5` (2 px wide). Slightly taller than
a citizen. Fades with daylight via alpha.

## properties that are deliberately *absent*

- no name in any string literal (grep the file:
  zero human-readable references)
- no tooltip, no click handler, no minimap icon
- no chronicle entry ever — verified post-trigger:
  `chronicle` tags remain `['milestone','misc']`
  across a full season cycle
- no `G.pilgrim` / `G.traveler` / `G.stranger` /
  `G.namedCharacters.*` entry — verified zero
  leaks to `G`
- no emoji. Citizens use emoji indicators
  elsewhere; this deliberately doesn't
- no sound cue
- no relation to any existing mechanic (happiness,
  trade, raids, research). The realm doesn't react
  to its presence
- not persisted. Reloading a save loses it — by
  design, the determinism is in the spawn key not
  in save state

## why this shape, not another

Considered and passed:

- **A rare bird that nests once per realm** — too
  easy to identify as "a bird"; naming would be
  trivial
- **A sound without a source** — would need audio
  work; also, the game is muted in most sessions
- **A tile that remembers what was built on it** —
  overlaps with the existing chronicle/story beats
  too much; the subtlety would get lost
- **A weather fragment** — already have rain,
  rainbows, mist; another weather layer is
  crowded
- **A rare color tint on one building** — overlaps
  with 045/047 graphics thread; too close to
  recent work to feel surprising

Picked the traversing silhouette because:

- the map edges have been unused since world-gen
  took its current shape
- motion draws the eye without a label
- "someone is here" is the simplest concept a
  player can form, and they have to form it
  themselves

## trade-offs

- **Player might miss it forever.** Not a bug. The
  challenge's whole point is optional content for
  the attentive.
- **Determinism vs feel.** I used a deterministic
  hash so same realm + season sees same path.
  Alternative was pure `Math.random()` — which
  would break reproducibility for future debugging.
- **Render ordering.** Registered as world renderer,
  so it draws between terrain and citizens. At
  tall-sprite overlap zones it might be occluded
  by a building; that's acceptable. The silhouette
  is tiny (2.4 × 9 px).
- **55% spawn chance.** Chose slightly-better-than-
  even so a new realm has a decent chance of
  seeing it in their first full year cycle but
  can still have a "dry" year.
