## the build

`docs/realm/js/story.js` — 12-line addition inside
`_nightmareImagesForState`, after the existing 4
named-character conditionals:

```js
// Loop 089: founder-conditional fragments, gated on 072's founders_named flag.
// Founders fire at day [3,6]; nightmare at [50,250] — so the flag will be set
// by the time nightmare fires, unless the player somehow skipped 072's window.
if (hasFlag('founders_named')) {
  const f1 = G.storyFlags.founder1;
  const f2 = G.storyFlags.founder2;
  const f3 = G.storyFlags.founder3;
  if (f1) pool.push(`${f1}'s footprints in fresh soil, leading only away from the realm`);
  if (f2) pool.push(`${f2} at the gate after sunset, not looking up`);
  if (f3) pool.push(`${f3}'s face on a stone that no hand in the realm carved`);
}
```

Pool grows 11-base → 14-base (or 15/18 if character fragments
also present). Three picks-of-three use existing slot-hash
logic; no changes to `checkNightmareBeat` itself.

## why this shape

- **Gated on `founders_named`, not individual founder flags** —
  matches 072's invariant that the 3 founder names are always
  set as a triple. Defensive `if (f1)` guards still present.
- **3 conditionals, 1 per founder** — same cardinality as
  072's founder slots. Each image is uncanny in a different
  way (footprints/gate/stone) so they don't collapse into
  parallelism.
- **Image tone matches existing static pool** — dream-logic
  surrealism (hand that didn't carve, footprints leading only
  away). Founders aren't villains; the nightmare just
  SURFACES them in uncanny composition.
- **Day-timing is safe** — founders fire [3,6] and nightmare
  [50,250]. The flag is essentially always set by nightmare
  time. The gate exists for defensive correctness, not
  expected runtime variance.
