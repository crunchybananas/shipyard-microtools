## the build

`docs/realm/js/render.js` — new exported function at
end of file (~65 lines including comment block):

```js
export function renderBuildingIsolated(type, opts = {}) {
  const width = opts.width || 100;
  const height = opts.height || 140;
  const daylight = opts.daylight ?? 1.0;
  const bgColor = opts.bgColor || '#4fb04f';
  const season = opts.season || 'spring';

  const off = document.createElement('canvas');
  off.width = width; off.height = height;
  const octx = off.getContext('2d');
  octx.fillStyle = bgColor;
  octx.fillRect(0, 0, width, height);

  const mockBuilding = {
    type, x: 0, y: 0, level: 1, hp: 100, maxHp: 100,
    workers: [], buildProgress: 1, productionTimer: 0, lastProduced: 0,
  };
  const s = { x: width / 2, y: Math.round(height * 0.72) };

  const origCamera = G.camera, origDayPhase = G.dayPhase, origSeason = G.season;
  G.camera = { x: 0, y: 0, zoom: 1.5 };
  G.dayPhase = (G.dayLength || 3600) * 0.5;
  G.season = season;
  try {
    drawBuilding(octx, mockBuilding, s, daylight);
  } finally {
    G.camera = origCamera;
    G.dayPhase = origDayPhase;
    G.season = origSeason;
  }
  return off;
}
```

`docs/realm/js/main.js` — 2 lines (import + debug
hook):

```js
import { ..., renderBuildingIsolated } from './render.js';
...
G.debug.renderBuildingIsolated = renderBuildingIsolated;
```

## why this shape

Considered and rejected:

- **Call `render()` with a mock G** — render draws
  the whole scene (sky, terrain, shadows, weather,
  enhancements, minimap). Isolating a single sprite
  means reimplementing the framing logic.
- **New module file `building_preview.js`** —
  couldn't reach `drawBuilding` without exporting
  it, which pollutes the public render.js API.
  Better to keep everything scoped to render.js.
- **OffscreenCanvas API** — in-thread faster but
  adds a second code path (OffscreenCanvas vs
  regular canvas). Vanilla `document.createElement
  ('canvas')` works identically and pops into the
  DOM trivially for debug (`appendChild`).

The G.camera/dayPhase/season swap is the only
state-mutation risk. Wrapped in try/finally so
any drawBuilding exception still restores state.

Chose `daylight = 1.0` default so the night-tint
overlay (render.js:2515+) doesn't fire — we want
a clean midday-bright sprite for silhouette/palette
work. Callers can pass `daylight: 0.5` to audit
dusk appearance.

Zoom set to 1.5 so:
- `camera.zoom >= 1.0` branches fire (worn-grass
  ring, night-window-glow, damage cracks)
- `camera.zoom >= 0.5` branches fire (snow caps,
  HP bars)

### 051 fix CONFIRMED

046 flagged house ≈ tavern as silhouette twins. The
profile confirms the blur was real AND confirms
051's flag fixed it:

- **house peakY = 64** (chimney at y-35 × 1.1 scale
  + sprite offsets = 64)
- **tavern peakY = 55** (new flag at y-36 × 1.1 + 3
  offsets, slightly above where house's chimney
  reaches)

**9-pixel peak-height delta.** That's a clear
silhouette differentiator — more than the 2-3 px
chimney-width deltas 046 named as the blur.

### bakery was already distinct

Surprising sub-finding: 046 ranked bakery in the
house/tavern cluster, but the profile shows
bakery's chunky chimney puts peakY at 43 — 21 px
above house. The cluster at default zoom may have
been *two* buildings (house + tavern), not three.
046's "three-way blur" finding was partly wrong;
only house and tavern were genuinely close.

Doesn't invalidate 051's fix — that broke the real
blur. But filing a correction: the 046 review read
source code from a "sprite primitive" level without
running the primitives. With renderBuildingIsolated
live, future silhouette-tests can ground-truth on
actual pixels.

## what I did NOT do

- **Export drawBuilding from render.js.** Would be
  the minimal change but pollutes the API. Kept
  `drawBuilding` module-private; the helper is the
  public surface.
- **Overlay grid or comparison UI.** One function
  returns one canvas. Composition (grid / overlay /
  diff) is the caller's job. Keeps the helper
  single-purpose.
- **Animate snapshots** (for buildings like windmill
  that animate). Single-frame is the common case.
  Animated review would need a different helper.
- **Apply the new profile data to rewrite 046.**
  046 is on the record; 052's correction is in
  this journal. Future ticks citing 046's findings
  should cross-check against 052.
- **Add a live-preview panel** that shows the
  isolated building in the game's HUD. Tempting
  but scope creep. Dev-only.
