## why stubs, not a full delete

The enhancements.js file has a culture of preserving
loop-number headers even when content retires. See
the pattern in the file header:

> DISABLING A FEATURE: add `return;` as the first
> statement inside the update/render function (NOT
> the registerXxx call). The register call stays so
> the ordering of later features isn't perturbed.

074 goes further than disable — it removes the
register call entirely, because the code was dead
from the start. But the section comment remains
so git blame + archaeology still works. Future
readers see "Loop 187 was a school-chronicle, now
retired as duplicate" instead of confusion about
a missing numbered loop.
