## the decision

**SHIP `maxCount: 1`** for townhall.

**Why one-shot, not tier-unlock:**
1. **Narrative coherence (load-bearing).**
   244 BUILDING_FIRST_BEATS prose ("the
   town hall opens. ${mayor} sits at the
   long table") and 253 mayor_first_in_hall
   ("a morning when ${mayor} unlocks the
   town hall... stands at the long window")
   both presume a SINGULAR town hall.
   Multiple townhalls breaks "THE long
   window" / "THE long table" — which one?
2. **Real-world coherence.** A town has
   ONE town hall. Multiple breaks the
   civic-governance metaphor that named the
   building.
3. **Named-character-gate semantics.**
   "Mayor unlocks THE townhall" reads as a
   one-shot reward, not a tier unlock that
   permits stacking.
4. **256 HIGH finding alignment.** The 257
   doc edit declared structural-unlock as
   "categorically different... boolean gate
   that delegates ongoing impact to the
   unlocked building." A SINGULAR unlocked
   building is consistent with that frame.
5. **Minimal blast radius.** First use of
   `maxCount` schema; defaults to absent
   for all other buildings (so no behavior
   change anywhere else).

## why this shape (architectural choices)

- **Generic, not townhall-special.**
  `isBuildingUnlocked` checks
  `def.maxCount != null` generically. Future
  capped buildings (if any) will work the
  same way without code changes here.
- **At `isBuildingUnlocked`, not `canPlace`.**
  The build-bar UI (ui.js:257) filters by
  `isBuildingUnlocked` — putting the gate
  there hides the BUTTON when at cap. UX
  payoff: player sees the button vanish
  after building townhall #1; no need to
  show + reject. (canPlace would still allow
  selection followed by rejection — strictly
  worse UX.)
- **Truthy check `!= null`, not `!== undefined`.**
  Allows future caller to pass 0 to
  effectively disable a building (rare, but
  semantically correct).
- **No retro maxCount on castle/church/etc.**
  The 256 finding noted that castle is
  similarly stackable; townhall has the
  STRONGEST narrative coherence argument
  (244+253 prose) for one-shot. Castle's
  "ultimate structure" framing is more
  ambiguous; defer that audit.
