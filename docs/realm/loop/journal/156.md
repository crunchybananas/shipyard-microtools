## why this shape

- **Grammar stated AT THE POINT OF USE.** 155
  captured the invariant in loop/docs/narrative-
  surfaces.md (the right place for global state),
  but the doc sits separately from the code. A
  future tick considering "should I acknowledge
  the owls?" reads story.js — putting the rules
  where the pattern is applied means the author
  sees them without doc-hunting.
- **Points at the canonical doc.** The comment
  says "Formalized as an invariant in loop/docs/
  narrative-surfaces.md per 155" — so the code
  is an inline summary, not a duplicate. If the
  doc changes, the comment stays stable; it's
  only a pointer.
- **Candidate list is explicit.** Owls / frogs /
  rams / trade-ships / hawks were filed at 152
  but the full list is now visible at the
  pattern site. Lower chance a future tick
  picks an already-filed candidate without
  knowing.
- **Queue-drain warning preserved.** The 155-
  filed concern (3-4 in rapid succession looks
  like queue-drain) is restated here as a soft
  constraint for future pickers.
- **No entry reordering.** The 148 and 152
  entries stay in the same file positions.
  Reordering to cluster misc-tag entries would
  shift surrounding diffs and risk git-blame
  confusion without changing behavior. The
  header comment is enough to signal grouping.
