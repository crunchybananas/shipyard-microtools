## the build

`docs/realm/js/story.js` — one new function
(`checkFounderBeat`) + one name pool + wired into
`checkStoryBeats`.

```js
const _FOUNDER_NAMES_POOL = [
  'Edda', 'Maren', 'Silas', 'Linna', 'Bram', 'Ivy', 'Torrin',
  'Hesper', 'Osric', 'Rhea', 'Corvin', 'Maia', 'Orin', 'Juno',
  'Wren', 'Thaddeus', 'Lira', 'Aldo', 'Bree', 'Finn',
];

export function checkFounderBeat() {
  if (hasFlag('founders_named')) return;
  if (G.day < 3) return;
  const kname = G.kingdomName || 'Realm';
  const targetDay = _dreamHash(`${kname}_founders_day`) % 4 + 3;  // [3, 6]
  if (G.day !== targetDay) return;
  if (G.dayPhase > 120) return;
  setFlag('founders_named');

  const pool = [..._FOUNDER_NAMES_POOL];
  const names = [];
  for (let i = 0; i < 3; i++) {
    const idx = _dreamHash(`${kname}_founder_${i}`) % pool.length;
    names.push(pool[idx]);
    pool.splice(idx, 1);
  }
  setFlag('founder1', names[0]);
  setFlag('founder2', names[1]);
  setFlag('founder3', names[2]);

  chronicle(
    `The three founders at last know each other's names: ${names[0]}, ${names[1]}, and ${names[2]}. The wicker packs are unpacked; the fire does not go out.`,
    'character'
  );
}
```

## design choices

- **Seeded day in [3, 6]** — not day 1 (the founding
  beat already owns that day) and not later than
  day 6 (names should land before the first-build
  beats cluster around day 2-5). The 4-day
  window spreads across realms.

- **20-name pool, 3 picks without replacement** —
  combinatorially gives each realm a unique triple
  (C(20,3) × 6 orderings ≈ 7,000 variants). Low
  collision rate across lifetime play.

- **Stored in `storyFlags` not `namedCharacters`**
  — the 034 `G.namedCharacters` structure has
  slots for mayor/bard/rival/smith/merchant/
  teacher. Founders aren't in that role taxonomy;
  adding them there would need a sub-structure.
  Flag storage is simpler and keeps future
  access pattern open.

- **Tag `character`** — auto echo-eligible (059/
  063 pool). A 200-day-old realm may hear "the
  bards still sing: 'The three founders at last
  know each other's names...'". The loop-back
  carries the founders' names into the realm's
  deeper lore automatically.

- **Prose echoes the founding beat** — "The
  wicker packs are unpacked; the fire does not
  go out" mirrors day 1's "Three weary settlers
  arrive at the shore, carrying all they own in
  wicker packs" and "three settlers at a fire
  that will not go out" (founding dream thread).
  Lyrical thread.

## what I did NOT do

- **Expose founder names on citizens**. The 3
  starting `G.citizens[]` entries get their own
  procedural names in `citizens.js`. Tying
  founder1/founder2/founder3 flags to specific
  citizens would require cross-referencing IDs.
  Out of scope; the chronicle entry is the
  narrative surface, citizen records don't need
  to care.
- **Nightmare-founder cross-reference**. Filed
  as a 072 follow-up — the nightmare image pool
  could add founder-conditional fragments like
  the mayor/smith/bard/teacher set.
- **UI display**. Founders don't need a panel
  tile. The chronicle entry IS the surface.
- **Variant beat texts**. One static beat is
  fine for a once-per-realm event (same pattern
  as 043 nightmare, 056 stone).
