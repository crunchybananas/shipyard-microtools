## the change

`docs/realm/js/main.js:loadAndStart` — 13 lines
added inside the `if (loadGame())` block:

```js
try {
  const ch = G.chronicle;
  if (ch && ch.length > 0) {
    const last = ch[ch.length - 1];
    notify(`Where we left off (day ${last.day}): ${last.text}`, 'info', { chronicle: false });
  }
} catch (_e) {}
```

## why this shape

- **Echo the chronicle, not a delta.** 127 shows "+2
  born, 1 raid survived" — a DELTA, because the game
  ran while the player was tab-blurred. 135 shows
  "day N: <text>" — an ECHO of the last beat, because
  the game was SAVED (no simulation ran during
  absence). Different information for different
  absence-types.
- **Last chronicle entry ≠ last day.** A realm saved
  on day 87 may have its last chronicle entry from
  day 84 (beats are sparse). Showing the last-written
  prose is more emotionally continuous than "day 87
  — <nothing happened>".
- **`chronicle: false`** — we're reading chronicle,
  not writing to it. Critical to avoid the read-
  then-write loop that would grow the chronicle every
  Continue.
- **`info` type** — same as 127. Non-intrusive toast.
- **try/catch** — defensive. If chronicle is malformed
  on a legacy save, we don't want loadAndStart to
  crash.
- **Scoped inside `if (loadGame())`** — only fires if
  load succeeded.
