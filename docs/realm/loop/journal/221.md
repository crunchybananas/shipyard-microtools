## the build

`docs/realm/js/render.js` — two changes.

### 1. Default flipped: `_USE_SVG_SPRITES = true`

The flag declaration moves from `false` to
`true`. Comment updated to reflect the
shipped-state. Console toggle preserved as
rollback path:
```js
window.__realm.toggleSVG(false)  // revert to canvas
```

### 2. Module-init eager preload

Added a `setTimeout(0)` block that calls
`_loadSprite(t, '')` for all 11 sprite types
at module init. Without this, the first frame
after game-start would show canvas-fallback
buildings for ~200ms while sprites loaded —
user-visible flicker.

The preload uses empty kname (default cache
key). Per-kingdom variants populate lazily on
first dispatch when `G.kingdomName` is set;
that's a one-frame canvas fallback per type
per realm, not per game-start.
