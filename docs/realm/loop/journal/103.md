## the build

`docs/realm/js/story.js` — three small changes:

### 1. New NARRATIVE_BEATS entry (closer of the table)

```js
{ flag: 'realm_fell', tag: 'requiem',
  trigger: G => G.day > 1 && G.population === 0,
  text: G => {
    const f = G.storyFlags.founder1;
    return f
      ? `The last fire in the realm goes out. ${f}'s name is the last spoken, and then there is no one to speak it.`
      : 'The last fire in the realm goes out. There is no one left to tend it, and no one left to remember.';
  } },
```

Placed at the end of NARRATIVE_BEATS, after the 3rd winter
beat. Composition-wise, the final table entry becomes the
realm's final narrative surface.

### 2. New tag `requiem` in TAG_ICONS

```js
requiem:'🕯️',  // Loop 103: realm-end beat. 1 tag, 1 beat (so far).
```

New tag justified per 075's anti-proliferation invariant
("if the subject is already a tag, reuse"): no existing tag
fits "the realm itself ends." `death` is for individual
deaths (combat, starvation). `victory` is oppositional.
`milestone` is positive. A new dedicated tag is right.

Icon 🕯️ (candle) matches the "last fire" imagery of the
prose. Also a traditional mourning/memorial symbol.

### 3. `requiem` added to `_EVICTION_IMMUNE_TAGS`

```js
const _EVICTION_IMMUNE_TAGS = new Set(['nightmare', 'stone', 'victory', 'requiem']);
```

Critical: when population drops to 0, a flood of starvation/
combat deaths writes to `death`-tag entries. If the realm
cap (300 entries) hits, cap-eviction could drop the realm-
fell beat itself before a player reads it. Adding `requiem`
to the eviction-immune set ensures this last-moment-of-the-
realm survives.

## why this shape

- **Compounds with 088/093/097 founder-arc.** Founder1 is
  canonical across 5 positive surfaces (first-snow,
  nightmare pool, 3 subsequent-winter beats, dream
  founding thread). 103 adds the 6th — the sad ending.
  "Lira's name is the last spoken, and then there is no
  one to speak it." The arc closes.
- **Uses NARRATIVE_BEATS (090 pattern).** No bespoke check
  function needed — the trigger+text+flag shape fits the
  existing table. Adds to the single-source invariant.
- **`G.day > 1` gate.** Defensive: prevents firing at game
  initialization if population state is briefly 0 before
  citizens spawn.
- **Trigger checks `population === 0` strict equality,
  not `<= 0`.** Should never be negative in practice, but
  strict equality communicates intent: "the population
  number BECAME 0."
- **Tag: new `requiem`.** New tags are unusual (075 anti-
  proliferation invariant), but the realm's own end
  deserves its own tag. Future extensions can reuse:
  castle falls, last founder dies, successor-less realm.
- **Eviction-immune.** 083/085's lesson: once-per-realm
  beats must be protected from cap eviction by death
  floods. 103 embraces this as a first-class gate.
- **No toast/notify.** The beat is the final chronicle
  entry; no player will interact with the game after it
  (population=0 = game-over). Adding a toast feels
  redundant when the game itself is ending.
- **Not added to `_ECHO_SOURCE_TAGS`.** Echoes fire at
  dawn, but post-realm-fall there are no more dawns.
  Requiem doesn't need to be echo-eligible.

## what this does NOT do

- **Trigger on castle destruction.** Castles can be
  damaged by raids but I didn't see player-accessible
  destruction. If a mechanic lands for castle-fall,
  another requiem beat could fire.
- **Trigger on time limit / realm abandonment.** If the
  player walks away for 100+ days with population stable,
  no requiem. That's correct — the realm isn't dead.
- **Reference other founders.** "Lira's name is the last
  spoken" works narratively because founder1 is the arc
  focal. If all 3 founders were named, "the founders'
  names are the last spoken" could work — but the
  individual form is more poignant.
- **Add post-requiem state** (e.g., a ruins render,
  `G.realmEnded` flag). Mechanical post-end behavior is
  out of 103's scope. Filed idea.
- **Add "realm forgotten" beat** — a second requiem that
  fires some days AFTER realm_fell. "The last chronicle
  entry is written. The book closes." Future tick.
