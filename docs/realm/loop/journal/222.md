## the hunt

Pattern from the 216 fix: prefix-numbered
identifiers (`_<name><loop-num>`) that were
intended as imports but never declared.

### Method

1. Across all 14 JS files in `docs/realm/js/`,
   `grep -oE "_[a-z][a-zA-Z]*[0-9]{2,3}"`
   to enumerate prefix-numbered identifiers.
2. For each unique ID, check if it has a
   declaration (`import`, `const`, `function`,
   `let`, `var`, `export`) OR is only used as
   a string value (quoted) OR is only in a
   comment.
3. Flag non-defined non-string usages.

### Results

- **enhancements.js** — 7 prefix-IDs total.
  All defined or comment-only. 216's fix
  holds: `_hf144`/`_sf144`/`_chr144` are
  imported as aliases at line ~5224.
- **economy.js** — 4 prefix-IDs
  (`_milestone10`/`25`/`50`/`75`). All are
  STRING values used as flag keys, not
  runtime identifiers.
- **story.js / combat.js / render.js /
  main.js / state.js** — no prefix-IDs (or
  all defined).
- **walkers.js / events.js / tech.js /
  audio.js / ui.js / notifications.js /
  trade.js / save.js / scenarios.js /
  achievements.js / animals.js / advisor.js**
  — all clean. No undefined prefix-IDs.
