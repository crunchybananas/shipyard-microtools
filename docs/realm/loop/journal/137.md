### 1. `docs/realm/js/audio.js` — new `namesake` case

```js
case 'namesake': {
  // F4 — first syllable
  makeOsc(ctx, dest, 'sine', 349, 0.045, 0.05, 0.4, t);
  // A4 — rising second syllable; starts as F4 fades
  makeOsc(ctx, dest, 'sine', 440, 0.050, 0.06, 1.2, t + 0.28);
  break;
}
```

**Sound design — spoken-name-coded:**
- **F4 (349 Hz)** at t, 0.4s sustain — the first
  syllable. Gentle, landing.
- **A4 (440 Hz)** at t+0.28s, 1.2s sustain — the
  rising second syllable. Starts as F4 fades.
- **Major 3rd** interval — bright without being
  triumphant.
- **Two-note phrase mimics** the rhythm of calling
  a name aloud. "Li-ra" or "Seph-i" — stress on
  second syllable, slight pitch rise.
- **Overlap mid-phrase** creates a subtle acoustic
  continuity — the two notes aren't separated by
  silence; they breathe into each other.
- **Gain 0.045 / 0.050** — quiet. The namesake
  beat's prose is "no one gathers, but no one stays
  still" — the sound should match that intimate
  scale, not broadcast.
- **Pure sines** — consistent with 111/113/115/125.

## why this shape

- **Name-call intonation** — every language with
  calling-to-person prosody uses a rising 2nd
  syllable. F4→A4 is the most universal version of
  that shape in tempered tuning. Culturally neutral
  (won't trigger a specific language's melody
  pattern).
- **Major 3rd, not minor** — child arriving is hopeful,
  not mournful. Even though the founder whose name
  passes on is AGING (085 wise-elder arc, 103
  last-spoken), the moment of namesake is BIRTH-
  centric, so major fits.
- **Shorter than offering (125's 1.8s)**, about the
  same as stone (113's 1.3-1.5s). A quiet personal
  moment, not a ceremonial one like founders (115's
  3-note triad with 0.9s per note).
- **Pattern 2 (`onFire`)** — namesake is in
  NARRATIVE_BEATS; naturally uses 123's refactor.
- **Doesn't duplicate any existing cue's
  coordinates.** F4 root is new; ascending major 3rd
  is new; "spoken name" structure is new. Passes
  112's distinct-fingerprint invariant.
