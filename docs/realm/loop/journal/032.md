# 032 — the-speedrunner (first use)

**Challenge:** the-speedrunner — accepted. First use.
Goal: measure minimum path to scenario completion and
surface friction a goal-oriented player encounters.
Also: break the 12/32 fixer cadence by picking a
fresh persona instead of marching toward the 40%
monoculture threshold.

Peaceful Valley picked because 029 simplified its
objective 2 — cleanest test of the scenario flow.

## methodology

Used chrome-mcp console to skip mouse/camera friction
and script the optimal path. This measures
*mechanical* minimum (what the game requires), not
*player* minimum (which adds pathfinding, camera
panning, error-correction). The gap between these
two is what I'm actually reporting.

Gave myself generous resources (500 wood/stone/food/
gold) to isolate the research-and-build flow from any
economy-gating. Research was set directly via
`G.researchedTechs.add(...)` to skip the 5+ minute
actual research wait — instrumentation, not a cheat.

## the minimum viable path (Peaceful)

**6 buildings + 4 techs**:

| Building | Why                                           |
| -------- | --------------------------------------------- |
| house #1 | +4 pop cap                                    |
| house #2 | +4 pop cap (total 11)                         |
| house #3 | +4 pop cap (total 15) — pops to cap           |
| church   | **objective 2** (requires masonry)            |
| well     | +5 happiness (requires engineering)           |
| tavern   | +10 happiness → 80% (requires brewing)        |

| Tech        | Prereq    | Time (ticks) | Unlocks          |
| ----------- | --------- | ------------ | ---------------- |
| masonry     | (none)    | 300          | wall, quarry, CHURCH |
| commerce    | (none)    | 350          | market, tradingpost, school, (enables brewing) |
| engineering | masonry   | 400          | road, WELL       |
| brewing     | commerce  | 250          | TAVERN           |

Research-only time: 300 + 350 + 400 + 250 = **1300 ticks**.
At speed 4× : ~325 real-seconds = **~5:25 minutes
just waiting for research** (sequential — research
queue is single-slot per `G.currentResearch`).

Build time + navigation time adds maybe 1–2 minutes
at speed 4. **Estimated real-speedrun time: 7–8 min.**

## happiness progression (measured)

From the happiness object:

```
After 3 houses + 1 church:  65%
+ 1 well:                   70%  (+5)
+ 1 tavern:                 80%  (+10 → objective complete)
```

Church contributed to the 65% baseline (its `desc`:
*"Major happiness boost for your settlement"*).

**This tells you exactly why a player would get stuck
at objective 3**. If they build only houses + church
(the most obvious mission-driven picks), they land at
65% and the game doesn't tell them which buildings
fill the remaining 15%.

## friction surfaced

### 1. No "how to get happiness" signposting

Objective 3 is *"Reach 80% happiness"*. The game has a
🙂 HUD icon that opens a happiness breakdown panel
(found via 027's audit), but:
- The objective itself doesn't link to it
- A speedrunner doesn't know to click the smiley
- No mission-panel hint like "boosted by wells,
  taverns, churches"

**Fix sketch**: extend the mission-panel objective
text on hover, or inline the happiness components
directly. Filing as idea.

### 2. Research is the hidden 5-minute bottleneck

A player who understands the path still has to wait
325 real-seconds at 4× for research queue to drain.
That's a single-session speedrun lower bound for
Peaceful. No research-booster building is available
at game start (School requires Commerce, so it's a
second-layer optimization).

**The School paradox**: research School to speed up
research. But researching School takes tech-time
itself. For a 4-tech Peaceful path, School isn't worth
it (adds one more research to save 50% on the rest).
For a 8-tech longer scenario like Industrial, School
probably IS worth it. No tick has measured this.

### 3. No speedrun timer

The game doesn't track elapsed play time or mission
completion time. A speedrunner can't see "you finished
in 6:42" — the only time metric is `G.day` and
`G.year`. That makes the scenario-complete moment
feel like a conclusion without a score attached.

**Fix sketch**: show play-time in the victory screen
and chronicle. Low effort, high speedrun-community
payoff.

### 4. The optimal path is legible to a player who reads

Every building's `desc` names its effect ("Major
happiness boost"). Every tech's `desc` names what it
unlocks ("Unlocks barracks and watch towers"). The
information is all there. A speedrunner who opens
Research + clicks through every building tooltip can
plan the optimal path in ~2 minutes of reading.

But the FIRST-TIME speedrunner doesn't know which
techs to research first. No in-game "recommended
order" exists; no tech tree visualization (the
Research panel lists techs but not as a dependency
graph). A hand-drawn tree would add significant
planning speed.

**Fix sketch**: arrange the Research panel as a
left-to-right dependency flow (masonry → engineering,
commerce → brewing → ...) with visual arrows. Non-
trivial UI work, not for this tick.

## what scripting skipped (honest)

My measured 1010ms "total time" is not a real
speedrun. The actual friction a player faces is:

- **Clicking through the build bar** to find each
  building (at 1.3× zoom the build bar is at the
  screen edge, some navigation required)
- **Camera panning** to find legal placement tiles
- **Dismissing notifications** to see the map
- **Reading the tutorial** (or dismissing it)
- **Waiting for research** (the 5-minute gate)
- **Reading happiness panel** to diagnose the 65% stall

Each of these is small but additive. A rough estimate
for a NEW player attempting a first-time speedrun of
Peaceful: 15–25 minutes. For a practiced runner: 7–10
minutes, mostly research wait.

## ideas generated → ideas.md

- Objective-panel hint text: on hover over a
  scenario objective, show a brief "how to achieve"
  tip. E.g., hovering "Reach 80% happiness" would
  show "Try building wells, taverns, churches."
  Pairs with Cluster D (onboarding).
- Elapsed-play-time in victory screen + chronicle:
  tracks `Date.now() - startedAt` or simply counts
  `G.gameTick`. Displayed at win as "You finished in
  X days / Y minutes real-time." Low effort.
- Research panel as a dependency graph: current list
  doesn't show "X requires Y." A visual flow (left-
  to-right with arrows) would speed up optimal-path
  planning. Non-trivial UI work.
- Speedrun mode scaffold: optional flag (`G.speedrun
  = true`) that starts the timer, disables saves (no
  cheating), surfaces the timer in HUD. Pairs with
  photo mode as a "modes" umbrella.
