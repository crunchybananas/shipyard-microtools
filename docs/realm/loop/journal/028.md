## what I recommend

Going forward, when a tick's findings coincidentally
close another tick's idea (like 027 → 002), the fixer-
or-play tick should proactively update ideas.md. The
archivist shouldn't be doing pattern-matching six ticks
after the fact.

Pattern to codify: **at the end of a [play] or [review]
tick, grep ideas.md for any line whose description is
narrower-than-or-equal-to the finding reported, and mark
DONE if applicable.** This turns the archivist from a
"cleanup" role into a "the last fixer did a polish pass"
discipline.
