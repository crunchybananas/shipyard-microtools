## why this shape

- **Trigger on `G.day`, not on year-math.** Old code computed
  `year = Math.floor((G.day - 1) / 28) + 1` then tested
  `year >= 2`. Equivalent to `G.day >= 29` but simpler.
  NARRATIVE_BEATS predicates are naturally per-field
  comparisons; importing year-math would leak logic up.
- **Day boundaries are:** year 2 @ day 29, year 3 @ day 57,
  year 5 @ day 113. Derived from `(N-1) * 28 + 1`. Computed
  once at migration, baked into the table. If realm calendar
  ever changes (28-day year is a magic number), 3 entries
  need updating — same number as before, just centralized.
- **Text-as-function for year2 (pluralization) and year3
  (kingdom name).** Year5 is a static string. All 3 cases
  already handled by the 090 table's `text: string|(G)=>string`
  shape. No schema change needed.
- **Conditional pluralization `${n === 1 ? '' : 's'}`** —
  matches 082's filed fix shape exactly. Single-character
  template expression; readable at the call site; no helper
  needed for one use.
- **Timing: NARRATIVE_BEATS runs per-`checkStoryBeats` (once
  per day/tick from main.js), old code ran every 120 ticks
  via registerUpdater.** Both fire at-most-once because of
  the `hasFlag` gate. New timing is actually slightly tighter
  — if a year boundary crossed mid-day, old code would wait
  for the next 120-tick sample; new code fires on the next
  checkStoryBeats call. Imperceptible difference; same
  firing day in practice.

## what this proves

- **NARRATIVE_BEATS works across files** — the pattern
  doesn't require the fire-source to be in story.js; any
  file's contribution just becomes a table entry. The
  implicit architecture is "story.js owns the beat
  registry; other files own the *triggers*, expressed as
  `(G) => bool`." Future cross-file additions follow the
  same pattern.
- **Migration is mechanically safe** — grammar + flag
  semantics preserved. The test suite verifies the tricky
  boundary cases (day=28 vs day=29; pop=1 vs pop=7).
- **Text-as-function generalizes.** castle (090), now year2
  and year3 — third and fourth uses of the pattern. Not
  gratuitous.
