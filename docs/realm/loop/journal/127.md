## the build

`docs/realm/js/main.js` — 50 lines: one `visibilitychange`
event handler with blur-snapshot + focus-diff-summary logic.

### blur path

```js
if (document.visibilityState === 'hidden') {
  if (G && G.day && !_blurSnapshot) {
    _blurSnapshot = {
      day: G.day, season: G.season,
      population: G.population,
      food, wood, gold,
      citizensBorn, citizensDied, raidsSurvived,
      wallTime: Date.now(),
    };
  }
}
```

Snapshots when page goes hidden. Guards on `!_blurSnapshot`
so nested blur→focus→blur→focus flickers don't trash the
original snapshot.

### focus path

```js
} else if (document.visibilityState === 'visible' && _blurSnapshot) {
  const snap = _blurSnapshot;
  _blurSnapshot = null;
  const daysPassed = G.day - snap.day;
  const wallMs = Date.now() - snap.wallTime;
  if (daysPassed < 1 && wallMs < 30_000) return;
  // ... compose parts, notify
}
```

On visible, compute deltas. Skip if absence was trivial
(<1 day AND <30 wall-seconds). Otherwise assemble a parts
array and fire a single toast via the existing `notify()`.

### composed summary

Parts included when non-zero:
- `N day(s)` (always)
- `→ <season>` if season changed
- `+N born` if births happened
- `−N lost` if deaths happened
- `N raid(s) survived` if raids happened
- `pop ±N` fallback if population changed but not via
  births/deaths (e.g., migration event)

Example output:
> "While you were away: 3 days, → winter, +1 born, 1 raid
> survived."

## why this shape

- **Passive information, not a prompt.** notify() with
  type='info' uses the toast channel — visible for 2.8s,
  no modal, no pause. Matches 126's "accept-the-loss
  design preserved" conclusion.
- **`chronicle: false`** — the summary is a UI artifact,
  not a narrative beat. Goes to the log badge / activity
  feed only. Chronicle stays clean.
- **30-second wall-time threshold** — prevents firing
  after a quick alt-tab. Player actually went AWAY.
- **1-day threshold** — covers the case where the
  player paused via focus loss but speed=0 (no game-days
  pass). Zero-day absences don't get summaries.
- **Wall-time snapshot** — not just day delta. A player
  at G.speed=0 who tabs away briefly won't generate days,
  but at G.speed=4 a 30-second absence = ~2 game-days.
- **Skip on no snapshot** — prevents double-fire if
  visibilitychange fires spuriously.
- **`_blurSnapshot` is module-scoped, not on G** —
  doesn't persist across save/load. Correct: absence-
  tracking is a SESSION concept, not realm state.
