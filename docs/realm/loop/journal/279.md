## why this shape

- **Castle roof, not castle stone.**
  Castle is THE most-prominent
  building (placed once; central;
  end-game). Roof color is the
  visually-loudest swappable element
  vs walls (which match 90% of stone
  buildings) or banners (which are
  small accents). Roof variation
  reads from far away.
- **4 palettes, not 2 or 3.** Castle's
  visual prominence justifies the
  full 4-variant ledger. Future small
  buildings (windmill, tower) might
  ship with fewer.
- **Faction-flavor names** in the
  comments (naval / moss / twilight)
  give future authors a hook for
  faction-text beats that align with
  the visual — e.g. blue-roof realm
  could reference "naval" themes in
  beats. Not enforced, just
  suggestion.
- **Day-hash via _kHash.** Same
  pipeline as church.glass + townhall
  .stone. No new infrastructure
  needed — 220 filing's pattern is
  now mature enough that adding
  variant entries is "fill in the
  table + run verify-variants."
- **Heraldic differentiation, not
  flavor variance.** The castle's
  ROOF feels like a faction marker
  (heraldry); the WALLS feel like
  shared architecture (stone). Same
  realm-language families share
  walls; different realms differ in
  banner/roof/sigil. This 279 entry
  encodes that intuition by varying
  only the heraldic surface.
