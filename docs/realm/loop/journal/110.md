# 110 — the-fixer (a11y viewport fix, closes 109 HIGH)

**Challenge:** the-fixer — accepted. Closes 109's HIGH
a11y violation: `user-scalable=no` in the viewport meta
was blocking native pinch-zoom for vision-impaired users.
1-line fix, same-day after-audit (109→110 pattern mirrors
095→096 and 107→108).

## the change

`docs/realm/index.html:5` — swapped attribute in the
viewport meta:

```html
<!-- before -->
<meta name="viewport" content="width=device-width,initial-scale=1.0,user-scalable=no">

<!-- after -->
<meta name="viewport" content="width=device-width,initial-scale=1.0,maximum-scale=5">
```

## why this shape

- **`maximum-scale=5`** keeps a cap on runaway zoom (e.g.,
  a fumbled two-finger gesture that stretches layout
  infinitely) while restoring the accessibility affordance.
  5× is WCAG's common recommendation — lets vision-
  impaired users enlarge substantially without losing the
  game's design bounds entirely.
- **No `user-scalable=no`** means modern iOS/Android
  default to "allow pinch zoom." This is what Apple's
  HIG recommends for non-specialized apps.
- **`initial-scale=1.0` unchanged.** First paint still
  renders at "normal" scale. Players who don't want to
  zoom see the exact layout they saw before.
- **Single character-class change.** The entire attribute
  value is the same shape; only `no` → `5` replaces one
  token and renames another (`user-scalable` →
  `maximum-scale`). Mechanical; unlikely to regress.

## what I did NOT do

- **Remove the meta entirely.** Dropping `content`
  defaults would lose `width=device-width` and risk bad
  zoom on first load.
- **Add an in-game accessibility-mode toggle.** 109 filed
  it as option (C); more scope than needed for a 1-line
  fix.
- **Test pinch-zoom live on a phone.** Extension
  disconnected; filed follow-up to verify on real device.
  CSS change is trivially verifiable though.
