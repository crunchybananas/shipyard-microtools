## the build

### 1. `js/events.js:278` — plague death (per 241)

```js

```

Inserted immediately after the existing
`stats.citizensDied++` so the partner stat
and tracker travel together.

### 2. `js/soldiers.js:145` — soldier combat (per 241)

Same single-line addition after the existing
citizensDied increment.

### 3. `js/economy.js:351` — starvation citizensDied (NEW finding)

While applying 241's parity-check, found the
INVERSE bug: economy.js starvation site sets
`lastDeathDay` but does NOT increment
`stats.citizensDied`. The partner stat was
missing from the OPPOSITE direction.

Added one line:
```js
if (G.stats) G.stats.citizensDied = (G.stats.citizensDied || 0) + 1;
```

This fixes a SECOND bug 241 didn't flag:
realms with starvation deaths but no other
deaths would report `citizensDied = 0`,
preventing the `no_death_known` beat from
firing AT ALL even after long peace (the
gate requires `citizensDied >= 1`).

separately would lose the lesson.
