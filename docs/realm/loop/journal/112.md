## why this shape

- **Mirrors narrative-surfaces.md** (075). Same structure
  (status, sources, philosophy, catalog, invariants,
  open ideas, related-loops, how-to-update). A reader
  who's read one doc knows how to navigate the other.
- **Captures THE PATTERN not just THE CODE.** Anyone can
  grep audio.js and find the two switch cases. The
  doc's value is documenting WHY they're shaped this
  way — the silent-miss-able philosophy, the contrast
  invariant, the "don't generalize until 3" rule.
- **Explicit "when to refactor" rule.** 2 tag-specific
  branches is coincidence; 3 is a pattern. Doc says this
  so future ticks don't prematurely ship an `onFire`
  callback field on NARRATIVE_BEATS before it earns
  its complexity.
- **Acoustic contrast table captured as invariant.**
  Future cues should produce DIFFERENT sonic
  fingerprints from existing ones. The grid lets
  authors see "where do I land on these axes?"
- **No code changes.** Pure doc tick. The audio code
  already works; this is just the instruction manual.
