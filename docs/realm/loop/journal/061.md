Closed by renaming `_t048` → `_wanderer` in

## the name

Considered (in order of increasing specificity):

- **the passerby** — too flat; implies indifference
- **the stranger** — implies danger, overshadows
  043's nightmare
- **the pilgrim** — religious overtone; realm has
  a church but no pilgrimage system
- **the messenger** — implies purpose; the element
  has none
- **the traveler** — neutral, close to right
- **the wanderer** — ✓

`wanderer` carries no destination, no authority, no
threat. Matches the element's behavior: crosses the
map without stopping, leaves no chronicle trace,
disappears at the far edge. A wanderer isn't going
*to* somewhere, they're just passing.

Also doesn't step on any existing named-character
space. The 6 named characters (mayor, bard, rival,
smith, merchant, teacher) are all rooted in a
building. A wanderer is rootless — distinct
category.

## why name the internal identifier but not the
## player-facing element

Deliberate. Two different audiences:

- **Code** needs a readable name so future ticks
  don't have to decode `_t048`. `the-name-giver`
  challenge specifically targets "confusingly
  named" — `_t048` qualified.
- **Player** still gets the silent element: no
  chronicle beat, no toast, no tooltip, no
  G.wanderer reference. The "content for the
  attentive" philosophy from 048 is preserved.

An attentive player who spots the element still
has to decide what to call it. A curious player
opening DevTools can now find `_wanderer` in
enhancements.js and understand what it is.

## what I did NOT change

- **The spawn-seed key** (`${kname}_048_${season}…`)
  — kept the "048" token so existing realms get
  the same spawn decisions they would have pre-
  rename. If someone loads a save mid-season, the
  same wanderer they'd have had fires. If I'd
  changed the seed string, spawn luck would
  diverge from saved state.
- **Any behavior.** No new chronicle beat, no
  new UI, no tooltip, no G reference, no trigger
  cross-refs. Pure name-only refactor.
- **The journal-historical references** to `_t048`
  in 048.md, 049.md, 050.md, 053.md. Those are
  record. The rename is only in live code.
- **Registration order.** Still
  `registerUpdater(_wandererTick); registerWorld-
  Renderer(_wandererDraw);` in the same place,
  between 023's volcano and 058's stone.
