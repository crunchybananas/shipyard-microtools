## why this shape

- **Static blades.** The 3D prototype renders
  static meshes (water animates via vertex shader;
  no per-frame mesh re-uploads). Animating
  windmill blades would require either per-frame
  mesh upload (slow) or a new per-vertex
  attribute marking "this is a blade vertex,
  rotate around hub Z" with a uniform sweep
  angle. Either is scope creep for a 2nd-touch
  tick. **Filed for a future improvement
  tick.**
- **Cross pattern at fixed angle.** 4 blades at
  0°/90°/180°/270° forms a crisp asterisk —
  reads as windmill at iso zoom even without
  rotation. Real windmills are often pictured
  in this canonical pose anyway.
- **Hub projection in +Z.** Hub cube extends
  from z=0.18 to z=0.30 (forward of the
  building center). Blades sit at z=0.20-0.24
  (forward of cap, slightly behind hub face).
  This puts blades clearly visible from iso
  angle without occluding the cap.
- **Reuses `pushCylinder` from 163.** Confirms
  the primitive is genuinely reusable across
  buildings — granary used it for silo;
  windmill uses it for stone base. 2 buildings
  + new primitive = pattern proven.
- **Single-radius cylinder, no taper.** Real
  windmills taper inward as they rise; canvas
  drawWindmill uses a trapezoid for this. The
  3D prototype lacks a frustum primitive
  (`pushFrustum(cx, cz, baseY, topY, baseR,
  topR, sides, c)`). Could add but keeps scope
  bounded for this tick. Filed.
- **Color palette borrowed from canvas.** Stone
  greys, wood browns, blade tan, hub
  near-black. Matches drawWindmill canvas
  intuition without being identical (3D
  prototype uses per-vertex flat color; canvas
  uses gradients — not directly comparable).
