## findings

### perceptually OK (cap sits correctly on roof)

1. **House** (x22, default baseline) — flat cap sits ON the
   peaked roof. Natural looking.
2. **Tavern** (x24, widest cap + 051 flag above) — cap sits
   at roof peak below the 051 flag. The flag rises above the
   snow line. Two-signal stack reads cleanly.
3. **Market** (x30, widest + flattest cap) — cap sits on the
   awning area, resembles a striped awning with snow settled
   on top. Reads as winter.
4. **Bakery** (x26, thicker cap, lower y) — minor floating
   but cap is close to the sprite's top. OK.
5. **Blacksmith** (x36) — peaked roof with chimney, cap sits
   neatly on peak. Looks natural.
6. **Barracks** (x38, slightly narrower, higher) — cap sits
   on the flat battlement top above the red flag-post.
   Works.

### perceptually floating (cap disconnected from building)

7. **Lumber mill** (x28, y=-29) — the chimney/peak rises to
   approximately the cap's vertical position, but there's a
   VISIBLE gap between the chimney top and the ellipse cap.
   The cap reads as sitting in mid-air just above the chimney.
   Moderate issue.
8. **Windmill** (x32, y=-32) — MOST severe. The cap floats
   significantly above the windmill's sail-cross (~4-6 px
   visible sky between the highest sail-tip and the cap).
   Clear perceptual disconnect. The cap DOES read as "on
   top of a windmill" because the axis is roughly aligned,
   but the gap is jarring.
9. **Granary** (x34, y=-26, wide ellipse) — the granary IS
   a dome. The wide ellipse cap sitting above the dome
   creates a visual doubling: "dome, then another dome
   above it." Reads as two snowy domes stacked. Worst
   offender visually — actually LESS elegant than pre-094.

## the tradeoff 094 made (explicit now)

094's config values prioritized **dimensional distinctness**
(width × height × yOffset triples all unique, verified via
052 pixel-diff). 095's perceptual check reveals the cost:
three buildings (windmill, granary, lumber) have y-offsets
that push the cap above their actual roof peak geometry.

The pre-094 baseline (y=-28 for all residentials) put every
cap at roughly the same height. That was PERCEPTUALLY safer
because it happened to coincide with most buildings' roof
lines. 094's y-axis variation ranges from -24 (market) to
-32 (windmill) — an 8-pixel spread that includes positions
above the actual sprite geometry for three buildings.

## specific fix recommendation

For follow-up tick, update `_WINTER_CAPS` in render.js:

```js
windmill:   { w: 9,  h: 3, y: -28 },  // was -32, revert to baseline
granary:    { w: 15, h: 3, y: -22 },  // was -26; lower to sit ON the dome
lumber:     { w: 11, h: 3, y: -28 },  // was -29; minor tweak
```

Width differentiation remains (windmill=9, granary=15, lumber=11
vs house=13 baseline), so pixel-diff distinctness is preserved.
y-offsets converge toward the visually-safe baseline. Losing
about half of the yOffset spread costs little silhouette-audit
value and pays the perceptual-coherence rent.

Alternative: 086-filed **dome-shape rendering** for bakery/
granary — instead of an ellipse cap, draw a shape that
conforms to the underlying dome. More code, better look.
