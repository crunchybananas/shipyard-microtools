## the build

### 1. `docs/realm/js/audio.js` — new `nightmare` case

Added to the `playSound` switch after the existing `chime`
case. ~10 code lines + comment block:

```js
case 'nightmare': {
  makeOsc(ctx, dest, 'triangle', 64, 0.10, 0.5, 2.2, t);
  makeOsc(ctx, dest, 'triangle', 68, 0.07, 0.5, 2.4, t);
  // High ghost overtone — barely audible, feels like a distant
  // bell no one remembered ringing.
  makeOsc(ctx, dest, 'sine',     384, 0.015, 0.3, 1.8, t + 0.15);
  break;
}
```

**Sound design:**
- **64 Hz** (C2) + **68 Hz** (flat C#2, +11 cents): minor-
  second interval. The two triangle waves at nearly-but-
  not-quite-the-same frequency produce audible BEATING at
  ~4 Hz — slow pulse that feels unstable.
- **384 Hz** sine overtone (5 octaves above the root, +
  phase offset): a distant bell-like ring that's easy to
  miss. Appears ~150ms after the chord begins.
- **Attack 0.5s** (long, sneaking in) + **decay 2.2s**
  (lingers). Total sound duration ~2.7 seconds.
- **Gain 0.10 peak** (quiet). A player absorbed in UI
  may not consciously hear it — which fits the "rarest
  moment" philosophy (nightmare once per realm; sound
  is correspondingly marginal).

### 2. `docs/realm/js/story.js` — fire sound from nightmare beat

Added a late-bound `_PLAY_SOUND` import mirroring the
existing `_NIGHTMARE_NOTIFY` pattern, plus a 3-line
`try { ps('nightmare') } catch` in `checkNightmareBeat`
after the existing toast:

```js
try {
  const ps = _PLAY_SOUND;
  if (ps) ps('nightmare');
} catch (_e) {}
```

Uses the same defensive load-order pattern as 043's
notify: import promise resolves at module load, local
variable holds the reference, try/catch around call
site protects from any late-load failure.

## why this shape

- **Dissonant but subsonic.** 64 Hz is below typical
  speech range; most laptop speakers reproduce it weakly.
  The 384 Hz overtone is what actually cuts through. The
  combination sounds like "something is wrong but you
  can't quite locate it" — which is exactly the nightmare
  beat's affect.
- **Beating effect at ~4 Hz.** 68 − 64 = 4 beats/sec. In
  music theory this is an unstable "wolf" interval. The
  ear hears a tremolo-like wavering. Different from a
  clean minor chord.
- **Late-bound import pattern.** story.js runs early in
  module graph; audio.js may load later. The existing
  NIGHTMARE_NOTIFY pattern works for this reason. 106
  adds a second variable (_PLAY_SOUND) parallel to the
  first.
- **try/catch on both.** Either notification or audio
  could fail to import (e.g., dev-server hiccup).
  chronicle(...) fires FIRST, so even with total audio/
  toast failure the beat still records.
- **No new tag, no new flag.** 043's `nightmare_fired`
  flag still gates the whole beat; sound inherits it.
  Playing the sound is a side-effect of the beat firing,
  not a separate gate.
