## why this shape

- **Combine into 1 tick.** Per 264
  [process] rule and 257 [process] rule:
  same-axis fixes that share a generator
  pattern combine. Both apply the 264
  variant-pool pattern; both target
  seasonal prose; both ≤30 LoC. Splitting
  into 265+266 would double overhead.
- **3 variants per season for transitions
  vs 4 for proverbs.** Transitions fire
  4× per year cycle (every 7 days × 4
  seasons); proverbs also 4× per year.
  Both need enough variation to NOT
  immediately cycle visibly. 3 variants
  × 4 cycles = 12 firings before
  guaranteed repeat (assuming day spread).
  4 variants × 4 cycles = 16 firings.
  Either is enough; differ slightly to
  stay non-uniform.
- **Day-based hash, not RNG.** Mirrors
  264. RNG breaks save/load
  reproducibility (same realm sees
  different prose across reload).
  Day-based selection: same realm sees
  same prose, two different realms see
  different prose only if their dayN
  hashes happen to collide (rare).
- **First variant preserves original
  prose.** Same as 264 backward-compat
  rule. Existing tests pass; saves with
  pre-265 chronicle entries don't show
  schema mismatch.
- **Two-sentence rhythm.** All 12 new
  season-transition variants follow the
  same TWO-SHORT-SENTENCES pattern as
  originals: "X. Y." Maintains the
  seasonal-stamp register without
  drifting toward elaborate prose.
- **Voice variety in proverbs.** New
  proverbs introduce new voices:
  "schoolteacher tells children" /
  "shepherd nods" / "miller stretches"
  / "baker mutters" / "elder rests on a
  bench" / "child asks the snow" /
  "elder by the fire". Voice rotation
  is the proverb-specific surprise —
  every new utterance has a slightly
  different speaker, never the same
  character giving the same advice.
