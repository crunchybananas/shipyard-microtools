### 3. Latent bug fix — `_hf144` / `_sf144` / `_chr144` undefined

`js/enhancements.js` line ~5224: 47 references
to `_hf144` / `_sf144` / `_chr144` across the
post-144 cluster (149 rival messages / 151 bard
songs / 152 mayor effects / 189 trading post)
had never been defined or imported. Every game
tick that hit one of these (~120-tick cadence)
threw `ReferenceError`, silently killing
half-a-dozen chronicle systems.

The intended pattern (per the existing
`chronicle as _chronicle124` import at line
4699) was to alias the helpers under a
144-suffix prefix. The import line was lost
somewhere between 144 and 189.

One-line fix: added the missing import block
at the start of the cluster:

```js
import { hasFlag as _hf144, setFlag as _sf144,
         chronicle as _chr144 } from './story.js';
```

Covers all 47 call sites. Verify-run after fix:
zero errors except favicon 404.
