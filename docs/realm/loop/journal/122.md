# 122 — surprise (un-filed: the stone weathers)

**Challenge:** surprise — accepted. Evolution mandate due

Extends 056's standing stone into a 3-beat lifetime arc:
- **056 discovery** (day 30-200): "A standing stone is
  found at dawn on the {direction} edge — {phrase}."
- **079 offering** (conditional on happy-peak + stone):
  "At dawn, a small blue stone the rivers have not seen
  was found at the standing stone."
- **122 weathering** (day ≥150): "Moss has begun to fill
  the name {kname} on the standing stone."

The stone is now the realm's longest-running
object-centered narrative — three beats across
~100-150+ days, each deepening the reader's sense of it
as a physical thing that persists.

## the build

`docs/realm/js/story.js:NARRATIVE_BEATS` — new entry
before first-snow. Tag: stone (reuse per 075 invariant).
4 variant texts picked by kingdom-hash:

```js
{ flag: 'stone_weathered', tag: 'stone',
  trigger: G => G.storyFlags.stone_found && G.day >= 150,
  text: G => {
    const kname = G.kingdomName || 'Realm';
    const variants = [
      `Moss has begun to fill the name ${kname} on the standing stone.`,
      `The stone leans a little more than it did. The name ${kname} remains.`,
      `Weather has written lines on the stone beside the name ${kname} — marks only time reads.`,
      `A bird has nested in the lee of the stone. ${kname}, it continues to say.`,
    ];
    return variants[_dreamHash(`${kname}_weather`) % variants.length];
  } },
```

## why this shape

- **Cross-system gate** — `stone_found` flag (from 056)
  + `day >= 150`. The stone must have been discovered
  AND ~100+ days must have passed. Realistic: the
  realm has LIVED WITH the stone long enough to observe
  change.
- **Tag: stone (reuse).** 056 owns this tag with 1 beat;
  079 owns it with a different beat at the same
  location. 122 adds a 3rd entry under the same tag.
  Three stone-tagged beats total, all about the same
  physical object. 075's anti-proliferation invariant
  is respected.
- **Eviction-immune.** `stone` is in
  `_EVICTION_IMMUNE_TAGS` (per 085). Late-game chronicle
  pressure won't drop this beat.
- **4 variants, kingdom-hashed.** Different kingdoms
  get different types of weathering. Moss, lean,
  weather-lines, bird-nest — each a different
  micro-story about how time has passed. Hash seed is
  `kname + "_weather"` — distinct from 056's `_stone_phrase`
  seed, so a kingdom's DISCOVERY phrase doesn't
  predict its WEATHERING phrase.
- **Kingdom name appears in all 4 texts.** Continuity
  with 056's invariant (kingdom name is ALWAYS the
  inscription).
- **Once-per-realm.** Same `hasFlag`/`setFlag` pattern
  as every other NARRATIVE_BEATS entry. Fires exactly
  once per realm when conditions meet.

## why day ≥ 150

- 056 stone fires seeded day [30, 200] — ~115 average.
- 079 offering fires ~15% probability on happy-peak
  dawn post-30. Average fire ~day 80 if happy realm.
- 122 weathering fires day ≥150 — so it's typically
  ~35+ days after 056 (at 056 median), ~70+ days after
  079 if offering fired.
- 150 is a realm-mid-to-late-game boundary; realms
  that don't reach day 150 (shorter plays, quick
  losses) never see the weathering — which is
  thematically correct. Only LONG-LIVED realms see
  their stones age.
