## why mutual exclusion matters

Each fallback gates on the original
beat's flag being UNSET. This prevents
double-firing for realms that:
1. Built church
2. Reached year3 → original beat
   fires (sets flag)
3. Church destroyed
4. Without mutex: fallback would ALSO
   fire → realm has BOTH
   "There is a step at the church
   door..." AND "The church is gone.
   The step is gone with it..."
   chronicling the same arc.

With mutex: only one of the two beats
fires per realm — whichever the realm
EXPERIENCED first (church-present
gives original; church-absent-at-y3
gives fallback).

Test #55 explicitly validates this.

## meta-observation

**Two-beat-with-mutex pattern**
established. Could be reused for
other building-gated beats with raid-
destruction risk. Pattern:

```
Original beat:
  trigger: year3 && building_present
  text: <building-present prose>

Fallback beat:
  trigger: year3 && everHadBuilding[type]
    && !building_present
    && !original_flag
  text: <building-absent prose with fresh angle>
```

**Pattern characteristics:**
- 2 NARRATIVE_BEATS entries per
  building-affected concept
- Mutual exclusion via flag check
- Fresh angle on fallback (not
  forced; per 257)
- Coverage: 100% for realms reaching
  y3 with everHadBuilding flag set

Could file as authoring pattern for
future building-gated beats. Not
universal — only worth the 2x-beat
overhead when the post-loss prose
genuinely surprises.
