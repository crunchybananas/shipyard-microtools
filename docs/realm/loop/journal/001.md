## why this shape

Prior realm loops (see `archive/LOOPS.md`, 200+ iterations) ran in a
single long session with 3 sub-agents (validator, divergent
thinker, implementer). That worked but drifted: later loops
sometimes re-proposed things earlier loops had tried, because
context compressed away.

This new shape copies discipline from `~/code/playground/strand`:

- **Fresh context every tick** via `ScheduleWakeup` — next
  Claude reads the MANIFEST cold, no carryover bias.
- **One challenge per tick**, chosen from a written pool —
  constraints that force off-the-obvious-path work.
- **Persona play** is first-class: half the challenges require
  actually loading realm in chrome-mcp and playing as
  someone-not-you. This catches UX problems that pure-code
  loops kept missing.
- **Append-only journal** with strict numbering. No more
  multiple overlapping log files (LOOPS.md + LOOPS_PROGRESS.md
  + LOOP_STATE.md + LOOP.md).
