# 134 — surprise (un-filed: a namesake)

**Challenge:** surprise — accepted. Evolution mandate
active (last surprise 128, 6 ticks back). Un-filed;
not in ideas.md.

Creates a new narrative surface: once per realm, a
child is born bearing a founder's name. Which founder is
deterministic per kingdom (hash-picked). Compounds 072
founder-naming + firstBirth — the realm's first
generation of continuity.

## the build

`docs/realm/js/story.js:NARRATIVE_BEATS` — new entry
added before 128's longest-night:

```js
{ flag: 'namesake_born', tag: 'character',
  trigger: G => {
    if (!G.storyFlags.founders_named) return false;
    const births = G.stats && (G.stats.citizensBorn || 0);
    return births >= 5;
  },
  text: G => {
    const kname = G.kingdomName || 'Realm';
    const idx = _dreamHash(`${kname}_namesake_who`) % 3;
    const key = ['founder1', 'founder2', 'founder3'][idx];
    const f = G.storyFlags[key] || 'the founder';
    return `A child is born in the realm and takes the name ${f}. The news carries from house to house — no one gathers, but no one stays still.`;
  } },
```

19 lines.

## why this shape

- **Cross-system compound** — requires 072 founders
  named + firstBirth pattern + 5 births minimum. Realm
  must have LIVED long enough to have reasonable
  population turnover before this fires.
- **Deterministic founder pick per kingdom** — each
  realm chooses ONE of its three founders as the
  canonical "whose name gets passed on." Like 116's
  constellation is deterministic per kingdom.
- **Hash seed `${kname}_namesake_who`** — distinct
  from all other hash seeds (088/089/093/097/116/122/
  128 use different keys). No cross-beat correlation.
- **Tag: character (reuse per 075)** — aligns with
  034's character intros. Another citizen joins the
  realm's named roster (implicitly).
- **"No one gathers, but no one stays still"** —
  voice matches 071 happiness beats ("Doors stand
  open; songs carry further than they should"). Quiet-
  but-significant register.

## the founder arc continues to rebalance

Cast before 134:
- founder1 (Lira) = 7 canonical surfaces
- founder2 = 4 surfaces
- founder3 = 4 surfaces

Cast after 134: depends on which founder 134 picks for
a given kingdom. For any individual realm, ONE of
the three gets a +1 surface (namesake). Across many
realms, the distribution balances: each founder gets
the namesake ~33% of the time.

Cumulative weight across many realms:
- founder1: 7 + ⅓ ≈ 7.3
- founder2: 4 + ⅓ ≈ 4.3
- founder3: 4 + ⅓ ≈ 4.3

Still founder1-heavy, but 134 doesn't worsen the
asymmetry.

distinct roles (field-stander, sky-watcher, night-
watcher). 134 adds the most generational role of
all — **their names outlive them**. A child bears
the name. The realm's first intergenerational
moment.

  trigger first goes true.

  named for Lira; a grandchild named for her." Very
  long-play scope. Filed.
