# 056 — a-scene-that-happens-once (the stone)

**Challenge:** `a-scene-that-happens-once` — accepted,
second take (043 was the first — NIGHTMARE). Evolution
mandate was 2 ticks past threshold (last surprise was
048, 7 ticks back; mandate asks 1-in-5). Re-took the
challenge with a fresh angle rather than picking
un-used challenges — surprise-shape matters more than
fresh-pool-name.

Shipped a new once-per-realm beat: a standing stone
found at dawn, inscribed with the kingdom's own name.
Deliberately parallel structure to 043's NIGHTMARE
but with different tone — nightmare is disturbance,
stone is discovery.

## the build

`docs/realm/js/story.js` — new `checkStoneBeat()`
function, added to the beat pipeline alongside
`checkDreamBeat` and `checkNightmareBeat`:

```js
export function checkStoneBeat() {
  if (hasFlag('stone_found')) return;
  const kname = G.kingdomName || 'Realm';
  const targetDay = _dreamHash(`${kname}_stone_day`) % 170 + 30;
  if (G.day !== targetDay) return;
  if (G.dayPhase > 120) return;
  setFlag('stone_found');

  const dir = _STONE_DIRECTIONS[_dreamHash(`${kname}_stone_dir`) % 4];
  const phrase = _STONE_PHRASES[_dreamHash(`${kname}_stone_phrase`) % 4](kname);
  chronicle(
    `A standing stone is found at dawn on the ${dir} edge of the fields — ${phrase}. No one remembers raising it.`,
    'stone'
  );
}
```

Four directional variants × four phrase variants =
16 prose forms. Each includes the kingdom name.

Phrase pool:
- "carved into its face is a single word: {kname}"
- "the kingdom's name is cut shallow into it: {kname}"
- "a name is pressed into the stone, barely legible — {kname}"
- "the wind has worn {kname} into the stone's lean"

## why this shape

- **Once-per-realm** matches 043's rarity cadence
- **Seeded day [30, 200]** — earlier than nightmare's
  [50, 250] so the stone lands before the nightmare
  for most realms. Player first discovers the stone,
  then much later the nightmare.
- **Dawn-only gate** (`dayPhase > 120 return`) matches
  039/043 — all three narrative beats share the same
  time-of-day rule.
- **No toast, no notification** — unlike the nightmare
  which fires 🌑 danger toast, the stone is a quiet
  chronicle entry only. Player who isn't watching
  the feed will see it as a callback when they open
  the chronicle panel later. Matches 048's philosophy
  of "content for the attentive."
- **Kingdom name is always inscribed** — the realm
  gets a canonical artifact of its own identity.
  Cross-references 024's kingdom-name system (the
  player named this thing; the world remembers).
- **No rendering this tick** — scoped purely to
  story.js. A future tick can add a visible stone
  sprite at a seeded tile; filed.

## what separates 043 nightmare from 056 stone

Deliberately kept distinct:

| aspect     | NIGHTMARE (043) | STONE (056)        |
|------------|-----------------|--------------------|
| day window | [50, 250]       | [30, 200]          |
| tag        | `nightmare`     | `stone`            |
| toast      | 🌑 danger       | none               |
| tone       | disturbance     | discovery          |
| character-conditional | yes (mayor/smith/bard/teacher) | no  |
| prose      | cascading list of 3 images | one complete sentence |
| chronicle weight | heavy (3 clauses) | light (2 clauses) |

The two beats should feel like different *categories*
of rare moment, not the same one retagged.

## verification (chrome-mcp on :8891 via 049 SW)

Tested 8 distinct kingdoms, scanning days 30..200
for each:

```
Avalon      → day 70   | western  | wind-worn
Brighthold  → day 166  | southern | cut-shallow
Cinderfen   → day 77   | eastern  | carved-single-word
Dragonspire → day 119  | northern | barely-legible
Ebonreach   → day 154  | southern | cut-shallow
Fairholm    → day 89   | northern | barely-legible
Greywater   → day 89   | northern | barely-legible
Hearthstone → day 86   | southern | wind-worn
```

- **7/8 unique days** — one collision at day 89
  (Fairholm & Greywater, different prose). ~12%
  collision rate across 170 possible days, within
  expected randomness.
- **All 4 directions surfaced**: northern 3,
  southern 3, eastern 1, western 1. Slight
  eastern/western under-sample, small-N artifact.
- **All 4 phrase variants surfaced** across the 8.
- **Determinism confirmed**: Avalon fired twice
  with identical prose (same seed).
- **No-refire confirmed**: flag prevents second
  fire within a realm.
- **Dawn gate held**: dayPhase=500 did not fire
  during the seeded-day scan.

`node --check js/story.js` → OK.

## what I did NOT do

- **Render the stone visibly.** A standing stone
  on a seeded tile would be the natural next step
  — visible artifact complementing the chronicle
  beat. Filed. Out of scope for a narrative-beat
  tick.
- **Call 048's unnamed element `the stone`.**
  They're deliberately different: _t048 traverses
  (ambient), stone persists in narrative (an
  artifact). Even the code keeps them separate.
- **Notify on discovery.** Considered a subtle
  ✴️ toast like "A discovery" but rejected — the
  stone's value is in being noticed, not announced.
- **Store seeded x/y coordinates.** Filed for
  when a render pass is added. For now, the
  direction ("northern edge of the fields") is
  the only spatial data and it's narrative-only.
- **Cross-reference named characters.** Unlike
  the nightmare which sources images from
  mayor/smith/bard/teacher, the stone uses ONLY
  the kingdom name. Keeps them structurally
  distinct and spares the already-rich
  named-character surface.
- **Adjust the cadence relative to nightmare.**
  The windows overlap (day 50-200 both fire);
  a realm could theoretically get the stone and
  nightmare on adjacent days. Left as-is; if it
  happens it'll produce a memorable cluster. Can
  tune window offsets later.
