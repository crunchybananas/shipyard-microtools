Realm SVG Sprite Sandbox

Loop 161 experiment. Sibling to render.js drawGranary (canvas implementation). The same building rendered as inline SVG with richer detail than the canvas primitives can produce, viewed at four sizes representing zoom levels in-game (48px ≈ default zoom; 384px ≈ photo-mode zoom).

Stage

What's in the SVG that the canvas version can't easily do:

  • linearGradient + radialGradient for the silo body and dome face — cleaner light falloff than canvas createLinearGradient chains, and they scale crisp at photo-mode zoom.
  • pattern for wood-grain on the dome — tileable, vector-clean.
  • Per-shape highlight + shadow strokes that don't pixel-stair.
  • Decorative grain specks, hinges, latch, finial detail — affordable to author at this fidelity in markup; expensive to maintain inside a procedural canvas function.

Open questions for future ticks:

  • Performance vs canvas at 100+ buildings on screen?
  • Replace drawGranary entirely or run side-by-side via a feature flag?
  • How does the SVG composite with the existing day/night tint and winter snow cap (158 dome-conforming arc)?
  • Should sprites be inlined into the bundle or fetched per-type?
